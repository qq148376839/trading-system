# 合并 Create/Edit 策略模态框为 StrategyFormModal

**日期**: 2026-03-14
**状态**: 已完成

## 背景

CreateStrategyModal（~800行）内联在 `page.tsx` 中，与独立组件 EditStrategyModal 功能高度重叠。两者 UI 风格不一致（create 用 antd Modal/Select/Input，edit 用原生 HTML），维护成本高。

## 方案

以 EditStrategyModal 为基准，合并为 `StrategyFormModal`，通过 `strategy?: Strategy` prop 区分模式：
- `undefined` = 创建模式
- 有值 = 编辑模式

## 7 个风险点处理

| # | 风险点 | 处理方式 |
|---|--------|---------|
| 1 | handleSubmit 分支 | `if (isEditMode && strategy)` 两套逻辑原封不动嵌入 |
| 2 | 模式切换清空 symbols | `symbols: isEditMode ? formData.symbolPoolConfig.symbols : []` |
| 3 | 资金计算 holdingValue | create 时 existingHoldings=[]，reduce 自然为 0 |
| 4 | useEffect 数据加载 | `if (strategy) promises.push(getStrategyHoldings(strategy.id))` |
| 5 | Institution 双重警告 | create 时 existingHoldings.length===0 自动走正确分支 |
| 6 | 期权参数 UI | 统一用 edit 的完整模板，create 由 DEFAULT_CONFIGS 提供默认值 |
| 7 | 说明卡片 BUG | 补上 SCHWARTZ 三分支；"不可修改"提示仅 edit 显示 |

## 关键设计决策

- **DEFAULT_CONFIGS 含 STANDARD preset 风控参数**: 期权类型创建时 config 自带 entryThresholdOverride/exitRules/rsiFilter/latePeriod 等完整默认值，presetMode 初始为 STANDARD
- **auto-detect CUSTOM**: useEffect 同时监听 OPTION_INTRADAY_V1 和 OPTION_SCHWARTZ_V1
- **create 模式类型切换**: 重置 config + symbols + preset + stockPoolMode + localNumbers
