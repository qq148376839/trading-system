## 一、设计决策摘要

| 决策点 | 结论 | 理由 |
|--------|------|------|
| 实施方式 | 替换 REVERSE_BEAR/BULL_SPREAD | 彻底改造，消除"无脑反向" |
| 架构 | 独立 MarketRegimeDetector 服务 | 关注点分离，不污染现有评分和策略评估逻辑 |
| 阈值 | 可配置，存 DB JSONB | 置信度低，需根据实盘数据迭代调整 |
| 对称性 | 同时处理看涨/看跌两个方向 | 数据证明 BULL 极端区同样失效（intradayScore > 15 时 25% 胜率） |
| UNCERTAIN 状态 | 保持原方向，仓位降至 50% | 不丢失交易机会，同时控制不确定性 |
| 当日动量确认 | 不纳入 | 极端阈值 + 分歧度已足够过滤；查库增加复杂度但边际收益低 |

| 前端 | 配置 + 日志展示 | 覆盖可配置性和可观测性 |

## 二、核心原理

### 2.1 根本问题

当前 `REVERSE_BEAR_SPREAD` 的逻辑：
```
score <= -spreadScoreMin → 无条件买 CALL
```
这是"无脑反向"——不区分市场处于动量延续还是均值回归状态。3/13 周五亏 $292 正是因为在 marketScore [-31, -35]（动量区间）触发了反向。

### 2.2 解决方案

在 finalScore 计算后、方向决策前，插入**市场状态判别层**：

```
当前:  score → threshold → direction → 策略评估 → 下单
改后:  score → threshold → MarketRegimeDetector → direction(可能被翻转) → 策略评估 → 下单
```

MarketRegimeDetector 判定三种状态：
- **MOMENTUM**（动量延续）→ 保持原方向，正常仓位
- **MEAN_REVERSION**（均值回归）→ 翻转方向，正常仓位
- **UNCERTAIN**（不确定）→ 保持原方向，仓位 × 50%

### 2.3 判定逻辑的第一性原理

**价格 = 供需均衡点。** 当一个方向的力量耗尽时，价格不需要新的力量就会向反方向运动。

判别力量是否耗尽的信号：
1. **分数极端度** → 力量已过度集中在一个方向
2. **分量分歧度** → 大盘和日内信号不一致 = 力量开始分化
3. **时间窗口** → 均值回归需要时间发挥，尾盘无空间

```typescript
export type MarketRegime = 'MOMENTUM' | 'MEAN_REVERSION' | 'UNCERTAIN';
export type RegimeConfidence = 'HIGH' | 'MEDIUM' | 'LOW';

export interface SmartReverseConfig {
  enabled: boolean;
  thresholds: {
    marketScoreExtreme: number;       // 默认 35（绝对值，对称）
    intradayScoreExtremeNeg: number;  // 默认 14（看跌极端阈值）
    intradayScoreExtremePos: number;  // 默认 15（看涨极端阈值）
    divergenceMin: number;            // 默认 0.3（分歧度阈值）
  };
  positionMultiplier: {
    reversed: number;                 // 反向交易仓位系数，默认 1.0（数据不足先不减仓）
    uncertain: number;                // UNCERTAIN 仓位系数，默认 0.5
  };
  timeWindow: {
    endHour: number;                  // 反向截止小时（ET），默认 14
    endMinute: number;                // 反向截止分钟（ET），默认 30
  };
}

export interface RegimeDetectionResult {
  regime: MarketRegime;
  confidence: RegimeConfidence;
  shouldReverse: boolean;
  positionMultiplier: number;        // 仓位系数 (0.5 ~ 1.0)
  reason: string;                    // 人类可读的决策原因
  metrics: {
    normMarket: number;              // 归一化 marketScore [-1, 1]
    normIntraday: number;            // 归一化 intradayScore [-1, 1]
    divergence: number;              // 分歧度
    marketExtreme: boolean;          // marketScore 是否极端
    intradayExtreme: boolean;        // intradayScore 是否极端
    withinTimeWindow: boolean;       // 是否在反向时间窗口内
  };
}

  enabled: false,                    // 默认关闭，需手动开启
  thresholds: {
    marketScoreExtreme: 35,
    intradayScoreExtremeNeg: 14,
    intradayScoreExtremePos: 15,
    divergenceMin: 0.3,

  positionMultiplier: {
    reversed: 1.0,
    uncertain: 0.5,
  },
```

**核心判定函数**:

```typescript
class MarketRegimeDetector {

  detectRegime(
    marketScore: number,      // 原始 marketScore（-100 ~ 100）
    intradayScore: number,    // 原始 intradayScore（-100 ~ 100）
    currentTimeET: Date,      // 当前美东时间
    config: SmartReverseConfig
  ): RegimeDetectionResult {

    // ---- 0. 未启用 → 直接返回 MOMENTUM（保持原方向）----
    if (!config.enabled) {
      return {
        regime: 'MOMENTUM',
        confidence: 'HIGH',
        shouldReverse: false,
        positionMultiplier: 1.0,
        reason: '智能反向未启用',
        metrics: { ...zeroMetrics }
      };
    }

    // ---- 1. 时间窗口检查 ----
    const hour = currentTimeET.getHours();
    const minute = currentTimeET.getMinutes();
    const timeMinutes = hour * 60 + minute;
    const endMinutes = config.timeWindow.endHour * 60 + config.timeWindow.endMinute;
    const withinTimeWindow = timeMinutes < endMinutes;

    if (!withinTimeWindow) {
      return {
        regime: 'MOMENTUM',
        confidence: 'HIGH',
        shouldReverse: false,
        positionMultiplier: 1.0,
        reason: `超出反向时间窗口(${hour}:${minute} >= ${config.timeWindow.endHour}:${config.timeWindow.endMinute})`,
        metrics: { ..., withinTimeWindow: false }
      };
    }

    // ---- 2. 归一化 + 极端度计算 ----
    const normMarket = marketScore / 40;     // 大致范围 [-1, 1]（raw score range ~[-40, 40]）
    const normIntraday = intradayScore / 20; // 大致范围 [-1, 1]（raw score range ~[-20, 20]）

    const marketExtreme = Math.abs(marketScore) > config.thresholds.marketScoreExtreme;
    const intradayExtreme = intradayScore < -config.thresholds.intradayScoreExtremeNeg
                         || intradayScore > config.thresholds.intradayScoreExtremePos;

    // ---- 3. 分歧度计算 ----
    const divergence = Math.abs(normMarket - normIntraday);

    // ---- 4. 状态判定 ----
    const metrics = {
      normMarket, normIntraday, divergence,
      marketExtreme, intradayExtreme, withinTimeWindow
    };

    // === 高置信反向 ===
    // marketScore 极端 + intradayScore 不极端 + 分歧度高
    // → 大盘过度反应但日内已企稳，均值回归概率最高
    if (marketExtreme && !intradayExtreme && divergence > config.thresholds.divergenceMin) {
      return {
        regime: 'MEAN_REVERSION',
        confidence: 'HIGH',
        shouldReverse: true,
        positionMultiplier: config.positionMultiplier.reversed,
        reason: `大盘极端(${marketScore.toFixed(1)})但日内企稳(${intradayScore.toFixed(1)})，分歧度=${divergence.toFixed(2)}`,
        metrics
      };
    }

    // === 中等置信反向 ===
    // 两个分量都极端且同向 → 全面过度反应，均值回归可能
    if (marketExtreme && intradayExtreme) {
      // 检查是否同向（都看跌或都看涨）
      const sameDirection = (marketScore < 0 && intradayScore < 0) || (marketScore > 0 && intradayScore > 0);
      if (sameDirection) {
        return {
          regime: 'MEAN_REVERSION',
          confidence: 'MEDIUM',
          shouldReverse: true,
          positionMultiplier: config.positionMultiplier.reversed,
          reason: `双极端同向(mkt=${marketScore.toFixed(1)}, intra=${intradayScore.toFixed(1)})，过度反应`,
          metrics
        };
      }
    }

    // === 仅 intradayScore 极端（marketScore 不极端）===
    // 日内过度反应但大盘未确认 → UNCERTAIN
    if (!marketExtreme && intradayExtreme) {
      return {
        regime: 'UNCERTAIN',
        confidence: 'LOW',
        shouldReverse: false,
        positionMultiplier: config.positionMultiplier.uncertain,
        reason: `仅日内极端(${intradayScore.toFixed(1)})，大盘未确认(${marketScore.toFixed(1)})`,
        metrics
      };
    }

    // === 默认：动量延续 ===
    // 中等区域，信号有效，顺势操作
    return {
      regime: 'MOMENTUM',
      confidence: 'HIGH',
      shouldReverse: false,
      positionMultiplier: 1.0,
      reason: `分数在中等区域，顺势操作`,
      metrics
    };
  }
}

export default new MarketRegimeDetector();
```

## 六、判定逻辑图

```
               ┌─────────────────────────────┐
               │  输入: marketScore, intraday │
               │  Score, 当前时间(ET)         │
               └─────────────┬───────────────┘
                             │
                    ┌────────▼────────┐
                    │ smartReverse    │  否
                    │ enabled?       ├──────► MOMENTUM (原方向, x1.0)
                    └────────┬────────┘
                             │ 是
                    ┌────────▼────────┐
                    │ 时间 < 截止时间? │  否
                    │ (默认 14:30 ET) ├──────► MOMENTUM (原方向, x1.0)
                    └────────┬────────┘
                             │ 是
               ┌─────────────▼─────────────┐
               │ 计算归一化分数 + 极端度    │
               │ normMkt = mktScore / 40   │
               │ normIntra = intraScore/20 │
               │ divergence = |normMkt -   │
               │              normIntra|   │
               └─────────────┬─────────────┘
                             │
            ┌────────────────▼────────────────┐
            │  mktExtreme && !intraExtreme   │  是
            │  && divergence > 0.3           ├──────► MEAN_REVERSION (反向, HIGH)
            └────────────────┬────────────────┘
                             │ 否
            ┌────────────────▼────────────────┐
            │  mktExtreme && intraExtreme    │  是 (同向)
            │  && sameDirection              ├──────► MEAN_REVERSION (反向, MEDIUM)
            └────────────────┬────────────────┘
                             │ 否
            ┌────────────────▼────────────────┐
            │  !mktExtreme && intraExtreme   │  是
            │                                ├──────► UNCERTAIN (原方向, x0.5)
            └────────────────┬────────────────┘
                             │ 否
```

## 八、向后兼容与迁移

### 8.1 旧策略配置兼容

现有数据库中可能存在 `strategyTypes.directional` 包含 `REVERSE_BEAR_SPREAD` 或 `REVERSE_BULL_SPREAD` 的策略配置。

**处理方式**: 在 `generateSignal()` 中，遍历策略时遇到 `REVERSE_*` 直接 `continue`（跳过），不报错。这样无需修改数据库中的旧配置。

建议后续通过前端编辑策略时自动清理这些旧值。

### 8.2 默认值策略

- `smartReverse.enabled` 默认 `false` → 现有策略不受影响
- 用户需要手动在前端开启智能反向
- 开启后，旧的 REVERSE_* 策略类型被忽略

## 九、阈值校准建议

当前阈值基于 41 笔（Week 2）+ 30 笔（Week 1，无反向数据）的分析结果，置信度低。

### 建议校准流程

1. **上线后收集 2 周数据**（约 50-70 笔），记录每笔交易的 regime detection 结果
2. **对比分析**：regime=MEAN_REVERSION 时反向是否真的更优
3. **调整阈值**：如果反向误触率 > 30%，放宽 marketScoreExtreme 至 40；如果反向正确率 > 60%，可收紧至 33
4. **分歧度 0.3 是最不确定的参数**：优先观察高分歧度（>0.5）案例的表现

### 阈值调整不需要改代码

所有阈值存在策略的 `config.smartReverse.thresholds` JSONB 中，前端可直接调整。

## 十一、风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
