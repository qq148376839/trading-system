# 大盘评分 USD/BTC/VIX 分K实时化 + 订阅推送

**日期**: 2026-04-04
**状态**: 已完成

## 问题

`calculateMarketScore()` 中 USD/BTC/VIX 只用日K数据：
- 日K反映 T-1 及更早的趋势背景，当日盘中状态完全是盲的
- USD 当天暴跌但 MA20 仍偏高 → 对股市施加错误的负向压力
- BTC 当天跟大盘共振上涨，日K还在反映昨天回调 → 共振信号延迟甚至反向
- VIX 盘中从 18 飙到 28 再回 20，日K收盘价完全错过恐慌峰值

SPX 已有分K（`calculateSpxMinuteTrend` 占 marketScore 20%），USD/BTC/VIX 需要同等对待。

## 方案

### 评分权重重分配

| 因子 | 改前 | 改后 |
|------|------|------|
| SPX 日线趋势 | 20% | 20%（不变） |
| SPX 分钟趋势 | 20% | 20%（不变） |
| USD 日线趋势 | 20% | 10% |
| USD 分钟趋势 | 无 | 10%（新增） |
| BTC 日线趋势 | 5-10% | 2-5% |
| BTC 分钟趋势 | 无 | 3-5%（新增） |
| VIX | 日K收盘 | 分K > 推送 > 日K |
| 市场温度 | 不变 | 不变 |

### 订阅推送架构

- LongPort: `subscribeCandlesticks(symbol, Period.Min_1)` → `setOnCandlestick` 回调 → 环形缓冲
- FutuOpenD: `CurKlineHandlerBase` → 环形缓冲 → `/realtime-kline` REST 端点
- 双通道订阅 SPY/UUP/IBIT，VIX 尝试分K失败降级报价
- 环形缓冲 120 bars（2小时 1min K线）
- 数据层优先读取订阅缓冲（>=15 bars），不足时回退 API 竞速

### VIX 三级数据源

1. 分K缓冲最新 close（来自订阅推送）
2. 推送报价 priceMap（实时报价订阅）
3. 日K最新 close（兜底）

## 验证要点

1. 部署后检查日志出现 `[QuoteSub] 大盘指数订阅完成`
2. futu-bridge 日志出现 `K 线实时订阅成功`
3. 盘中 `[大盘评分明细]` 日志确认 USD/BTC 分钟趋势字段
4. VIX 盘中波动时使用实时价而非昨收
5. `curl http://futu-bridge:8765/realtime-kline?symbol=US.SPY` 返回缓冲数据
