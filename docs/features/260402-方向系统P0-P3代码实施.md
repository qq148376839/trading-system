## P0: TIME_STOP 退出修复

### 改动内容 (`option-dynamic-exit.service.ts`)

1. **亏损下界**: TIME_STOP 条件增加 `pnl.grossPnLPercent >= -10`
   - 深度亏损 (< -10%) 交给 STOP_LOSS 处理
   - TIME_STOP 只管"theta吃死钱"场景 (PnL 在 -10% ~ +5% 之间)

2. **Delta-based 动态 maxHold**:
   - `|delta| > 0.6`: effectiveMaxHold = max(config, 25min) — 深度ITM，theta慢
   - `|delta| < 0.3`: effectiveMaxHold = min(config, 10min) — 深度OTM，theta快
   - 其他: 使用配置值

3. **策略10配置建议**: maxHoldMinutes 15 → 25 (需通过前端UI修改)

### 4/1 NVDA 回测效果

```
旧: 15min时PnL=-18.7% → TIME_STOP触发 → 错失$510
新: 15min时PnL=-18.7% < -10% → TIME_STOP不触发
    → STOP_LOSS在-20%待命(但NVDA反弹,未触发)
    → TRAILING_STOP接管 → 预计退出价~$1.80-2.00
```

## P2: timeWindow 改阈值修正

### 核心改动

**旧公式**:
```
finalScore = marketScore × 0.2 + intradayScore × 0.6 + timeWindowAdjustment × 0.2
```

**新公式**:
```
finalScore = marketScore × 0.2 + intradayScore × 0.8
effectiveThreshold = directionalScoreMin × vixFactor × timeThresholdFactor
```

### timeThresholdFactor 曲线

| 时段 | 因子 | 效果 |
|------|------|------|
| 9:30-10:30 (开盘1h) | 0.85 | 降低阈值，更易入场（但不偏向方向） |
| 10:30-12:00 | 1.0 | 基准 |
| 12:00-14:00 | 1.0→1.3 | 逐步收紧 |
| 14:00-15:00 | 1.3→1.8 | theta加速 |
| 15:00-15:30 | 1.8→2.5 | 强烈收紧 |
| 15:30+ | 3.0 | 几乎不入场 |

### 消除CALL偏置的效果

旧: 开盘时 timeWindow=+20 → finalScore += 4.0 → 系统性推CALL
新: timeThresholdFactor=0.85 → 阈值 9×0.85=7.65 → CALL和PUT同等对待

## P3: marketScore 多窗口 + Gap

### 多窗口趋势分析

替代原始的单一20日均线偏离：

| 窗口 | 计算 | 特点 |
|------|------|------|
| 3天 | (current - avg3) / avg3 | 短期动量，捕捉转折 |
| 10天 | (current - avg10) / avg10 | 中期趋势 |
| 20天 | (current - avg20) / avg20 | 长期趋势（原始方法） |

窗口权重由趋势一致性动态决定：

| 场景 | d3 | d10 | d20 | 说明 |
|------|-----|-----|-----|------|
| 全部同向 | 0.3 | 0.4 | 0.3 | 强趋势确认 |
| 短期反向 | 0.7 | 0.2 | 0.1 | 转折点，短期主导 |
| 混合 | 0.5 | 0.3 | 0.2 | 默认 |

### 3/31 回测效果

```
旧(20日均线): trend20d = -2.8% × 5 × 0.4 = -5.6 → 推PUT → 错误(SPX +1.73%)
新(多窗口):   trend3d=+2.5%, trend10d=-1.4%, trend20d=-2.8%
              sign(d3)=+ ≠ sign(d20)=- → 转折 → d3权重0.7
              trendScore = 2.5×0.7 + (-1.4)×0.2 + (-2.8)×0.1 = +1.19
              → marketScore正值 → 不推PUT → 正确!
```

### Gap 信号

- 检测 `(todayOpen - prevClose) / prevClose`
- |gap| < 0.3% → 忽略
- gap 线性映射到 ±15 分（0.3% → 2分, 1% → 8分, 2% → 15分封顶）
- 作为 marketScore 的补充分量
