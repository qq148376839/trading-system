# 交易分析系统实施方案

**日期**: 2026-03-14
**功能**: 新增交易分析页面 + 期权K线持久化采集服务

---

## 概述

新增 `/quant/analysis` 交易分析页面，包含综合分析、每日明细、反向分析、K线图表四个 Tab。同时新增期权K线数据采集服务，定时在收盘后采集当日期权订单对应的**正向 + 反向**期权K线数据并持久化到数据库，避免期权过期后无法获取历史K线。

### 背景

从周分析报告(260314-周交易分析.html)的实践来看：
- Week 2 (3/9-3/13) 的期权K线在 3/14 报告生成时仍可获取
- Week 1 (3/3-3/6) 的期权已完全过期，Longbridge API 无法返回K线数据
- **结论**：期权过期后K线数据会在一定时间后不可获取，正向和反向都需要及时保存

## Phase 2: 后端服务 + API 路由

### 2A. 新建 `api/src/services/option-kline-collection.service.ts`

核心类 `OptionKlineCollectionService`（单例模式，参照 `kline-collection.service.ts`）：

**关键方法：**

1. **`deriveReverseSymbol(symbol: string): string | null`**
   - 使用 `parseOptionSymbol()` (`api/src/utils/options-symbol.ts:95`)
   - 交换 C↔P，重构 symbol

2. **`collectForDate(date: string)`** -- 采集指定日期
   - 查询已成交期权订单（`tradeCtx.historyOrders()`）
   - 过滤期权订单（`isLikelyOptionSymbol()`）
   - 配对买卖订单
   - 对每笔交易：同时采集**正向K线 + 反向K线**
   - 请求间隔 800ms（Longbridge 限流保护）

3. **`collectForOrder(params)`** -- 单笔采集
   - 推导反向 symbol
   - 检查 analysis summary 是否已 SUCCESS → 跳过
   - 用 `quoteCtx.historyCandlesticksByOffset()` 获取：
     - **正向K线**：原始期权 symbol 的1分钟K线
     - **反向K线**：反向期权 symbol 的1分钟K线
   - 批量 upsert 到 `option_trade_kline`（ON CONFLICT DO NOTHING）
   - 计算 analysis summary 字段
   - 更新 summary 状态

4. **`init()` / `scheduleDailyCollection()`** -- 定时调度
   - 每日 US 收盘后 ~16:05 ET 触发
   - 使用 `getMarketCloseWindow()` + `setTimeout`

5. **`stop()`** -- 清理定时器

### 2B. 新建 `api/src/routes/option-kline.ts`

## Phase 3: 历史数据补采

### 首次部署后的一次性操作

上线后需立即执行一次 3/9-3/13 的K线采集，避免数据过期丢失：

```bash
# 通过 API 手动触发
curl -X POST http://localhost:3001/api/quant/option-kline/collect \
  -H "Content-Type: application/json" \
  -d '{"dateRange": {"start": "2026-03-09", "end": "2026-03-13"}}'
```

或在前端分析页面提供"补采历史数据"按钮。

**验证步骤：**
1. 先手动测试单个已过期期权（如 `TSLA260309P382500.US`）是否能获取K线
2. 如果能获取 → 批量采集 3/9-3/13 全部订单
3. 如果不能获取 → 说明 3/14 当天仍能获取但更晚就不行了，需要立即采集

### 4C. 新建 `frontend/app/quant/analysis/page.tsx`

**页面结构：**

```
AppLayout > Card
├── 标题栏 + 策略筛选 Select + 日期范围选择器 (RangePicker)
├── 摘要卡片 (Row > Col > Card > Statistic) × 6
│   总盈亏 / 有效交易 / 胜率 / 平均盈利 / 平均亏损 / 交易天数
│
├── Tabs (antd Tabs 组件)
│
│   ├── Tab 1: 综合分析
│   │   ├── 策略对比卡片 (按 strategy_id 分组，点击行可筛选)
│   │   ├── 方向/Regime 分析卡片 (CALL/PUT + 反向/不确定标签)
│   │   ├── 退出方式分析卡片 (止盈/止损/追踪止损)
│   │   ├── 每日盈亏瀑布图 (Recharts BarChart 正负双向)
│   │   ├── 得分 vs 盈亏散点图 (Recharts ScatterChart)
│   │   └── 得分区间胜率分析表格
│   │
│   ├── Tab 2: 每日明细
│   │   └── 按日分组明细 (含策略名/方向/Regime标签/得分/盈亏)
│   │
│   ├── Tab 3: 反向分析
│   │   ├── 总览对比卡片 (原始 vs 反向 PnL/胜率)
│   │   └── 逐笔反向对比表格
│   │
│   └── Tab 4: K线图表
│       ├── 交易选择器 (Select 下拉)
│       └── 双线图 (原始 + 反向期权K线, Recharts LineChart)
```

**多策略支持：**
- 信号日志 (`strategy_signals`) 的 `strategy_id` 用于关联策略
- 策略列表通过 `GET /api/quant/strategies` 加载
- 顶部策略 Select 筛选 + 策略对比表格行点击筛选
- 所有统计（摘要/图表/明细）均受策略筛选影响

**智能反向策略集成：**
- 信号的 `metadata.regimeDetection` 提供 regime 信息
- 方向分析卡片按 regime 状态分组（顺势/反向/不确定）
- 每日明细显示 regime 标签（反向=红色 Tag, 不确定=橙色 Tag）

**数据流：**
- Tab 1-2：`/api/quant/signals` (含 strategy_id + metadata) + `/api/quant/strategies` → 前端计算
- Tab 3-4：`/api/quant/option-kline/analysis` + `/api/quant/option-kline/candles`

**样式规范（参照 HTML 报告暗色主题）：**
- 暗色卡片：`background: '#161b22'`, `border: '1px solid #30363d'`
- 盈利 `#3fb950` / 亏损 `#f85149` / 中性 `#d29922`
- 策略标签：BULL 绿 / BEAR 红 / REV_BEAR 紫
- 退出标签：止盈绿 / 追踪止损黄 / 止损红

**移动端适配（Rule #11, 375px）：**
- `useIsMobile()` 控制图表高度、表格列显隐、卡片网格
- 表格 `scroll={{ x: 'max-content' }}`
- 摘要卡片 `xs={12} sm={8} md={4}` 响应式
- RangePicker 全宽
