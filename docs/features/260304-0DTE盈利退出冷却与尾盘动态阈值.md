# 0DTE 盈利退出冷却 + 尾盘动态入场阈值

**日期**: 2026-03-04
**触发**: 3月3日实盘数据 — 同底层换行权价秒级重入全亏 + 15:30后密集开Put被theta秒杀

## 改进 1：0DTE 盈利退出后冷却加倍

### 问题
- 冷却仅基于 `consecutiveLosses` + `dailyTradeCount`
- 盈利退出时 `consecutiveLosses=0`，冷却仅 1 分钟（第 2-4 笔）
- SLV 74P→71P、TSLA 385P→380P，盈利退出后秒级换行权价重入，二次进场全亏

### 方案
在两处冷却计算块中，`dailyTradeCount <= 3` 的无连亏分支，检测 `lastTradePnL > 0` 时冷却从 1 分钟提升到 3 分钟。

### 修改文件
- `api/src/services/strategy-scheduler.service.ts`
  - `processSymbol` 冷却块（~L1964）
  - `evaluateIdleSymbol` 冷却块（~L2428）

### 日志标记
- 冷却日志新增 `_profitExit` 后缀：`COOLDOWN_3m_trade#2_profitExit`

## 改进 2：0DTE 尾盘动态入场阈值

### 问题
- `option-recommendation.service.ts` 的 `calculateTimeWindowAdjustment()` 已有时间惩罚，但只占 finalScore 的 20% 权重，不足以阻止弱信号
- `schwartz-option-strategy.ts` 的 `getEntryScoreMin()` 无时间维度
- 15:30 后 QQQ/SPY 仍能以 -15 分入场，0DTE theta 加速衰减导致 4-7 分钟被止损

### 方案
在**阈值侧**引入时间衰减因子，与评分侧的惩罚形成双重收紧：

| 时段 (ET) | timeFactor | Schwartz 阈值 (base=12) | Recommendation 阈值 (base=15) |
|-----------|-----------|------------------------|------------------------------|
| 9:30-12:00 | 1.0 | 12 | 15 |
| 12:00-14:00 | 1.0→1.3 | 12→16 | 15→20 |
| 14:00-15:00 | 1.3→1.8 | 16→22 | 20→27 |
| 15:00-15:30 | 1.8→2.5 | 22→30 | 27→38 |
| 15:30+ | 3.0 | 36 | 45 |

### 修改文件
- `api/src/services/strategies/schwartz-option-strategy.ts`
  - `getEntryScoreMin()` — 乘以 `timeFactor`
  - 新增 `getTimeDecayThresholdFactor()` 方法（仅 0DTE 生效）
- `api/src/services/option-recommendation.service.ts`
  - 方向判定阈值从固定 `15` 改为 `15 * entryTimeFactor`
  - 新增 `get0DTETimeThresholdFactor()` 方法（同曲线）

### 日志标记
- Schwartz 得分不足日志新增 `timeFactor=X.XX`
- Recommendation 方向判定日志新增 `阈值=XX.X timeFactor=X.XX`
