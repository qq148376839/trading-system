# 时区管理统一化 — 修复周末下单 Bug

**日期**: 2026-03-23
**优先级**: P0（资金安全）
**状态**: ✅ 已修复

---

## 问题描述

3月22日（周六），系统在非交易日提交了 8 笔订单，全部未成交。

### 根因分析

系统已有三级交易日判断，但每级都存在时区 bug：

```
策略调度器 → isInTradingSession(market)
  ├─ 1级: Longbridge tradingSession() API → 周末返回空数组 ✓
  │   └─ 但时间比较用 getHours() 提取本地时间 ✗
  ├─ 2级: Longbridge tradingDays() API
  │   └─ NaiveDate 构造用 getFullYear()/getMonth()/getDate() ✗
  │   └─ 周末快速过滤用 getDay() ✗
  └─ 3级: getDay() 周末兜底 ✗
```

**核心问题**: `getDay()` 返回的是服务器本地时区的星期几，而非市场所在时区。Docker 容器未显式设置 `TZ`，继承 NAS 宿主机的 UTC+8 时区，导致：

- UTC 周五 16:00（美东 11:00 AM，交易日）→ 北京时间周六 00:00 → `getDay()` 返回 6 → 误判为周末
- UTC 周六 00:00（北京时间 08:00，中国周六）→ `getDay()` 可能返回 5（取决于 TZ）→ 误判为工作日

同类 bug 散布在 **13 个文件、59 处调用**中。

## 修复方案

### Phase 1: 统一时区模块 + 关键 Bug 修复

#### 1. 新建 `api/src/utils/market-time.ts`

创建统一时区模块，作为所有市场时间操作的唯一入口。基于 `Intl.DateTimeFormat.formatToParts()` 实现，自动处理 DST：

| 导出函数 | 用途 |
|----------|------|
| `getMarketTimeZone(market)` | 唯一时区映射源（US→America/New_York） |
| `getMarketLocalDate(date, market)` | UTC → 市场本地 year/month/day |
| `getMarketLocalTime(date, market)` | UTC → 市场本地 hour/minute/second |
| `getMarketLocalDayOfWeek(date, market)` | 市场本地星期几 (0=Sun) |
| `isWeekend(date, market)` | 市场时区周末判断 |
| `formatAsYYYYMMDD(date, market)` | 市场本地日期 → "YYYYMMDD" |
| `zonedTimeToUtc(params)` | 市场本地时间 → UTC Date（DST 安全，迭代收敛） |
| `toNaiveDateParts(date, market)` | Longbridge NaiveDate 构造用市场本地日期 |

#### 2. 修复 3 个关键 weekend check

| 文件 | 修复前 | 修复后 |
|------|--------|--------|
| `trading-session.service.ts` | `now.getDay()` | `isWeekend(now, market)` |
| `trading-days.service.ts` | `date.getDay()` | `isWeekend(date, market)` |
| `utils/trading-days.ts` | `date.getDay()` | `isWeekend(date, market)` |

#### 3. Docker TZ 固定

- `docker-compose.yml` → app service 加 `TZ: UTC`
- `docker-compose.dev.yml` → api service 加 `TZ: UTC`
- `Dockerfile` → `ENV TZ=UTC`

#### 4. 重构已有时区代码

- `market-session.service.ts` — 删除内部 `getMarketTimeZone`/`getLocalYMD`/`zonedTimeToUtc`，改为从 `market-time.ts` 导入
- `trading-session.service.ts` — 删除私有 `getMarketTimeZone()`/`getLocalTime()` 方法，改为导入
