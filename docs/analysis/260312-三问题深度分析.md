## 2. 为什么只买1张合约？——资金分配机制深度分析

核心问题：权利金 $1.43，1张合约 = $143.63，如果组1有 $1000 可用应该能买 6 张。

**实际机制**（`option-intraday-strategy.ts:767-799`）：

```
budget = min(availableCapital, maxPositionPerSymbol)

maxPositionPerSymbol = min(
  allocatedAmount / survivorCount,
  allocatedAmount × maxConcentration
)
```

**策略总资金 = $7000（配置名"7000刀期权"）**

关键变量 `survivorCount` = 竞价胜者数 + 当前 HOLDING 标的数。以 IWM 入场为例（15:06:04）：

- `allocatedAmount = $7000`（两组总计）
- 入场时 IDLE=6, HOLDING=0, 候选=1 → `survivorCount = 1`
- 但实际计算时还要考虑：
  - `maxConcentration = 0.33`（33%）
  - `maxPositionPerSymbol = min(7000/1, 7000*0.33) = min(7000, 2310) = $2310`

这本应给出 $2310 的预算，但日志显示：

```
[IWM.US/BEAR_SPREAD] PUT IWM260312P246000.US | 合约=3, 权利金=$0.72, 预估成本=$217.84
```

IWM 只买了 3 张（$217.84）而不是理论上限的 ~32 张！

**问题在于 `availableCapital`**。看资金差异告警：

```
15:05:13 [资金差异告警] 严重差异 - 记录值 0.00, 实际值 249.00, 差异 249.00 (12.45%)
14:00:13 [资金差异告警] 严重差异 - 记录值 412.00, 实际值 206.00, 差异 206.00 (10.30%)
```

**策略记录的可用资金与实际严重不匹配**，记录值经常接近 0 或远低于实际。当 `availableCapital ≈ $200-250` 时：

```
budget = min(~$250, $2310) = ~$250
```

这就解释了为什么每笔只买 $143-$272 的合约——**不是分组限制了预算，而是 `availableCapital` 的记录值本身就偏低**。

从日志可以看到每次平仓后释放的资金只有 $143-$272，这些被释放的金额又立即被下一笔交易消耗掉。系统像是在"滚动"使用一小部分资金，而不是用满 $7000 的分配。

**根因推测**：
- 策略 $7000 总资金被分为多个标的的占用
- 但每次只有 1 个标的 HOLDING，释放后可用资金只恢复了那一个标的的份额（~$200-270）
- 其余资金可能被"锁死"在其他标的的预留中，或资金账本不同步

## 3. 同标的快速重入场——分组机制分析

日志完整揭示了分组互斥运作机制：

### 分组配置（从日志推断）

- **组1（INDEX_ETF）**: TQQQ, QQQ, SPY, IWM
- **组2**: TSLA, ORCL, NVDA

### 同组互斥正常工作的案例

- TQQQ 持仓时 → `QQQ.US 被同组 HOLDING 标的 TQQQ.US 阻止`、`IWM.US 被同组 HOLDING 标的 TQQQ.US 阻止` ✓
- ORCL 持仓时 → `TSLA.US 被同组 HOLDING 标的 ORCL.US 阻止`（从 14:02 到 14:40 持续阻止） ✓
- QQQ 持仓时 → `SPY.US 被同组 HOLDING 标的 QQQ.US 阻止` ✓

### 跨组 floor 连锁保护也在工作

- QQQ 平仓后 → `SPY.US 被跨组 floor 连锁过滤 (group=INDEX_ETF, exitSymbol=QQQ.US, 1682s ago)` ✓

## 核心问题总结

| 问题 | 根因 | 建议 |
|------|------|------|
| 每笔只买1-3张合约 | `availableCapital` 记录值远低于实际，系统只用 ~$200-250 在交易 | 排查资金账本同步问题，$7000 的大部分资金去哪了 |
| 同标的秒级重入 | 组互斥只管不同标的，同标的平仓后无冷却期 | 为非0DTE配置 `zdteCooldownMinutes`（至少3-5分钟） |
| QQQ 自己平仓后自己重入 | 跨组 floor 保护了同组其他标的（SPY），但没保护退出标的本身 | 退出标的本身也应纳入 floor 冷却 |
