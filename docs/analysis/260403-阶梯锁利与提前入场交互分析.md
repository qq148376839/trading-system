# 阶梯锁利 × Phase 4 提前入场交互分析

**日期**: 2026-04-03
**起因**: TSLA 260402 P365 Trade #1 — 峰值+12.8%，实际锁利退出+2.2%，担忧 Phase 4 提前入场是否加剧此问题

---

## 1. 事实复盘：TSLA P365 Trade #1 到底发生了什么

| 指标 | 数据 |
|------|------|
| 入场 | 13:53 UTC, $2.07, 4张 |
| 峰值 | ~+12.8%（$2.34 附近） |
| 退出 | 13:54 UTC, $2.12, +2.2% |
| 退出标签 | `TRAILING_STOP` |
| 后续走势 | 40min后同一合约价格 $1.59（进一步下跌） |

**关键事实**：退出后 put 价格从 $2.12 降到 $1.59，说明 TSLA 反弹了。这笔交易的退出**事后看是正确的** — 不是"错过大盈利"，而是"及时保住了利润"。

---

## 2. 第一性原理分析：退出机制的数学冲突

### 2.1 当前退出机制叠加（0DTE EARLY 时段）

三层退出同时生效，各自独立判断：

```
Layer 1: 止盈            pnl >= 40% (配置) → 退出
Layer 2: 阶梯锁利        peak >= 8% → floor 2%; peak >= 12% → floor 6%  → pnl ≤ floor → 退出
Layer 3: 0DTE 追踪止损   peak >= 8% → drawdown >= 10-12% → 退出
```

### 2.2 谁先触发？

以 Trade #1 为例（peak=12.8%, current=2.2%）：

| 退出机制 | 触发条件 | 判定 |
|----------|----------|------|
| 阶梯锁利 | pnl ≤ 6%（12%台阶的 floor） | **先触发**：2.2% < 6% ✓ |
| 追踪止损 | drawdown ≥ 10%（12.8-2.2=10.6%） | **也触发**：10.6% ≥ 10% ✓ |

代码优先级：阶梯锁利（L472）在追踪止损（L501）**之前**检查。所以实际触发的是阶梯锁利。

但日志标签显示 `TRAILING_STOP` — 因为阶梯锁利也返回 `action: 'TRAILING_STOP'`（只是 exitTag 不同：`ratchet_profit_lock` vs `trailing_stop`）。

### 2.3 两个退出线的实际位置

peak = 12.8% 时的退出线：
- **阶梯锁利**：exit at ≤ 6%（保留 6% 利润）
- **追踪止损**：exit at ≤ 2.8%（peak - 10% = 12.8 - 10 = 2.8%）

**阶梯锁利更激进** — 它在 6% 就退了，追踪止损会等到 2.8% 才退。

问题不在追踪止损，**问题在阶梯锁利的台阶设计**。

---

## 3. 核心问题：0DTE 高 Gamma 与阶梯锁利的数学矛盾

### 3.1 0DTE 期权的波动特征

0DTE 期权 Gamma 极高，PnL% 变化非常剧烈：
- 标的移动 $1 → 期权可能变化 10-20%
- **15秒内** 10%+ 的 PnL 摆幅是常态，不是异常

### 3.2 当前阶梯设计

```
{ threshold: 8, floor: 2 }    → gap = 6%
{ threshold: 12, floor: 6 }   → gap = 6%
{ threshold: 18, floor: 12 }  → gap = 6%
{ threshold: 30, floor: 22 }  → gap = 8%
{ threshold: 50, floor: 38 }  → gap = 12%
```

**低台阶的 gap 只有 6%**。对 0DTE 来说，6% 的回撤空间在 30 秒内就能消耗完。

### 3.3 典型场景：正常走势中的"假退出"

```
T+0: 入场 $1.50
T+1: $1.65 (+10%) → 踩上 8% 台阶, floor=2%
T+2: $1.53 (+2%)  → 正常回调，仍在趋势中
T+3: $1.52 (+1.3%) → 低于 floor 2%?? → 退出！
T+5: $1.85 (+23%) → 已经不在了
T+8: $2.20 (+47%) → 完全错过
```

**6% 的回调空间无法容纳 0DTE 的正常波动**。趋势中的正常呼吸就能触发阶梯锁利。

---

## 4. Phase 4 提前入场如何改变这个局面

### 4.1 有利：PnL% 基数更大

提前入场 = 入场价更低 = 同样价格点的 PnL% 更高：

```
不提前: 入场 $1.50, 峰值 $1.65 → peak = +10%
提前3min: 入场 $1.20, 峰值 $1.65 → peak = +37.5%
```

提前入场后 peak 高得多，可能直接跳到更高的台阶（30%→22% floor），回调空间从 6% 扩大到 15.5%。

### 4.2 不利：加速构建期的振荡

Phase 4 在动量**正在构建**时入场，这个阶段价格振荡频繁：

```
T+0: 入场 $1.20（加速度 bonus 触发）
T+0.5: $1.31 (+9.2%) → 踩上 8% 台阶, floor=2%
T+1: $1.22 (+1.7%) → 低于 floor 2% → 退出！
T+3: $1.50 (+25%) → 已经不在了
```

**提前入场恰好落在振荡区间**，第一个台阶的 2% floor 极易被击穿。

### 4.3 净效应判断

| 场景 | 提前入场影响 | 阶梯锁利影响 | 综合 |
|------|------------|------------|------|
| 趋势启动 + 正常回调 | +（PnL%高，台阶容忍度大） | -（仍可能被低台阶截杀） | **看回调幅度** |
| 趋势启动 + 深回调 | +（PnL%高，floor更高） | ≈（可能survive） | **净正** |
| 假突破 + 反转 | -（更早暴露在假信号中） | +（低floor及时止损） | **净平** |
| 强趋势无回调 | ++（完整吃到大段） | ≈（不触发） | **净正** |

**结论：Phase 4 提前入场 + 当前阶梯锁利的组合，在"趋势启动+正常小幅回调"场景下有被截杀风险。核心矛盾在第一台阶 {8%, 2%} 的 6% gap 太窄。**

---

## 5. 定量验证：8% → 2% 台阶对 0DTE 的命中率

0DTE ATM 期权 Gamma ≈ 0.15-0.25，标的每 $1 移动 ≈ 期权变动 $0.15-$0.25。

以 SPY 为例（期权价 ~$2.00）：
- 标的 $1 移动 → 期权 ≈ $0.20 → PnL ≈ 10%
- **标的 $0.60 反弹**（SPY 常见的 1-2min 噪声）→ 期权 ≈ $0.12 → PnL 回撤 ≈ 6%

**SPY $0.60 的反弹在开盘 30 分钟内几乎每分钟都在发生**。

这意味着：一旦 PnL 峰值触碰 8%（标的移动 ~$1），紧接着的任何 $0.60 噪声回弹都会把 PnL 打到 ≤2%，触发退出。

## 6. 修复方案评估

### 方案 A：放宽低台阶 gap（推荐）

```typescript
// 当前
{ threshold: 8, floor: 2 }    // gap=6% — 0DTE 噪声即可触发

// 建议：0DTE 使用更宽松的第一台阶
{ threshold: 12, floor: 2 }   // gap=10% — 容纳 0DTE 正常波动
{ threshold: 18, floor: 8 }   // gap=10%
{ threshold: 30, floor: 20 }  // gap=10%
{ threshold: 50, floor: 38 }  // gap=12%（不变）
```

**原理**：第一个台阶从 8% 提高到 12%，确保是"真正的趋势利润"而非噪声波动。gap 从 6% 扩大到 10%，覆盖 0DTE $0.60 级别的正常回弹。

### 方案 B：阶梯锁利加入预热期

入场后 N 分钟内不启用阶梯锁利，只依赖追踪止损。

缺点：增加复杂度，且追踪止损在低 peak 时退出更晚（2.8% vs 6%），可能导致利润回吐更多。

### 方案 C：阶梯锁利按 0DTE/非0DTE 分别配置

```typescript
const PROFIT_LOCK_STEPS_0DTE = [
  { threshold: 12, floor: 2 },
  { threshold: 20, floor: 10 },
  { threshold: 35, floor: 25 },
  { threshold: 50, floor: 38 },
];
const PROFIT_LOCK_STEPS_NON0DTE = PROFIT_LOCK_STEPS;  // 保持现有
```

**优势**：0DTE 和非0DTE 的波动特征完全不同，分别配置更合理。
**缺点**：当前所有交易都是 0DTE，非0DTE 分支暂无实际数据验证。

## 7. 结论

| 问题 | 判定 |
|------|------|
| Trade #1 的退出是否错误？ | **不是** — 退出后 put 确实继续贬值，方向判断已失效 |
| 阶梯锁利是否会导致过早退出？ | **是** — 第一台阶 {8%, 2%} 对 0DTE 太紧，6% gap 不够 |
| Phase 4 是否加剧这个问题？ | **部分** — 提前入场拉高 PnL%（有利），但加速构建期振荡（不利） |
| 是否需要修复？ | **是** — 但不是因为 Trade #1，而是因为 0DTE 的 Gamma 特征决定了低台阶必须更宽 |
| 优先级？ | **建议观察 Phase 4 上线后 `ratchet_profit_lock` 退出的频率，再决定是否修改** |

**Phase 4 加速度 bonus 本身的设计是对的** — 更早入场确实能捕获更大的利润空间。阶梯锁利的窄 gap 是独立问题，不应该因此推迟或撤回 Phase 4。
