# 0DTE 单腿（裸 Long Put/Call）动态风控交易方案（V1）

**生成时间**：2026-02-16

**适用范围**：长桥（LongPort）仅支持单腿下单（LO/MO/LIT/MIT/TSLPAMT/TSLPPCT），当前策略实际执行为**单腿裸 long option**（并非真实价差/组合单）。

**目标**：在“单腿 + 末日（0DTE）”的现实约束下，实现 **少做更稳、回撤可控、长期期望值（EV）为正**。

---

## 核心结论（为什么要这样做）

- 当前命名为 `BEAR_SPREAD` / `BULL_SPREAD` 的策略在实现层面是**单腿裸 long**，暴露于 **Delta + Gamma + Theta**。
- 0DTE 的主要亏损形态往往来自：**开盘噪声被放大 → 底部/顶部入场 → 1-3 分钟 V 反转 → 期权价格快速崩塌**；只靠固定止损百分比很难稳定。
- 因为无法做“定义风险价差”（无组合单、卖方权限/保证金不确定），稳定性要靠：
  - **交易窗口选择（开盘禁 0DTE）**
  - **入场确认更苛刻（评分稳定 + 结构确认）**
  - **动态退出（结构失效/时间止损优先，PnL 兜底）**
  - **追踪止盈（锁住顺风段）**

---

## 术语与输入数据

- **score**：策略综合得分（例如 `finalScore` 或等价的触发分数）。
- **评估频率**：约每 5 秒一次（以当前系统为准）。
- **1m K 线**：标的 1 分钟 OHLCV。
- **VWAP**：标的当日成交量加权均价（可计算或直接获取）。
- **期权价格**：优先使用 `mid = (bid + ask)/2`，其次 `lastDone`（成交价），避免跳价误判。

---

## 方案总览（最终落地规则）

### 1) 交易窗口（Trade Window）

- **09:30-10:00 ET**：**禁止 0DTE 新开仓**
  - 允许：**1DTE/2DTE 单腿**（若仍满足入场规则）
  - 目的：规避开盘噪声 + V 反转对 0DTE 的致命影响

- **10:00-10:30 ET**（若你仍坚持 `firstHourOnly`）：
  - 允许 0DTE，但需满足更严格的入场与动态退出规则

> 备注：如果后续保留 `direction_confirm`（前 30 分钟方向确认），它仍可以作为额外过滤层，但**不应替代**本方案的 0DTE 开盘禁入与结构确认。

---

### 2) 入场规则（Entry）

#### 2.1 0DTE 入场阈值（你已确认）

- **PUT（看跌）入场**：`score ≤ -12`
- **CALL（看涨）入场**：建议对称：`score ≥ +12`

> 如果你暂时只做 PUT，也可以仅启用 `score ≤ -12`。

#### 2.2 评分稳定确认（Consecutive Confirm）

- 在满足阈值的前提下，要求 **连续 N 次评估**同向且达标：
  - **默认 N=2**（≈10 秒）
  - 若 `score` 在最近 1-2 分钟波动很大（跳变明显），提高到 **N=3**（≈15 秒）

目的：避免“短促极值”把信号打穿阈值而误触发。

#### 2.3 结构确认（Structure Confirm，基于 VWAP + 1m）

对每个标的 **各自判断**：

- **PUT 入场结构**（示例规则）：
  - 标的 **连续 2 根 1m 收盘价在 VWAP 下方**
  - 且最近 3 根 1m 不出现“强反转”特征（例如：
    - 1m 大阳线从低点拉升并接近/穿越 VWAP
    - 或单根 1m 涨幅/实体超过当日早盘分钟波动的高分位）

- **CALL 入场结构**对称：
  - 连续 2 根 1m 收盘价在 VWAP 上方
  - 且无“强下砸反转”特征

> 这条是专门用来规避 02-13 那种“入场接近绝对低点、随后 3 分钟 V 反转”。

---

## 3) 动态退出（Exit）——三层优先级（结构 > 时间 > PnL）

> 对 0DTE 单腿，**优先用结构与时间止损**，PnL% 只做最后兜底。

### 3.1 Level A：结构失效止损（最高优先级）

- **PUT 持仓**：若标的出现“做空结构失效”，立即平仓。
  - 推荐触发：**收盘价连续 2 根 1m 站回 VWAP 上方**

- **CALL 持仓**对称：
  - 收盘价连续 2 根 1m 跌回 VWAP 下方

> 结构失效往往比期权 PnL% 更早、更可靠地提示“方向错了/反转了”。

### 3.2 Level B：时间止损（0DTE 特有）

逻辑：入场后若没有快速走出顺风，Theta + 反抽会迅速吞噬 EV。

- 定义一个观察窗 **T 分钟**：
  - 若在 T 分钟内没有出现“最小顺风延续”，则退出

**“最小顺风延续”建议满足其一**：
- 标的方向继续走出新低/新高（PUT：新低；CALL：新高）
- 或期权 `mid` 回到 **≥ +5%**（小幅盈利即可，表示顺风）

#### T 的动态化（按当日 09:30-10:00 的 1m 波动分桶）

对每个标的单独计算：

- `rangePct = (High_0930_1000 - Low_0930_1000) / Open_0930 * 100%`

建议分桶（可调）：
- **高波动日**：`rangePct ≥ 0.65%` → **T = 3 分钟**
- **中波动日**：`0.45% ≤ rangePct < 0.65%` → **T = 5 分钟**
- **低波动日**：`rangePct < 0.45%` → **T = 8 分钟**

> 波动越大，V 反转越常见，0DTE 越要“更快认错”。

### 3.3 Level C：PnL 兜底（最后防线）

- **临时兜底阈值（你已接受）**：0DTE 使用 `mid` 计算的 **亏损 ≤ -25%** 触发退出
- 保留系统级硬安全阀（例如 **-40%**）作为最后断尾

> 重要：若当前系统存在“3-10 分钟止损放宽到 52.5%”的逻辑，建议对 0DTE **禁用放宽或反向收紧**，否则容易直接滑落到硬阀才离场。

---

## 4) 止盈：追踪止盈（TSLPPCT）

你选择“追踪止盈”，推荐做成“两阶段”：**先触发，再追踪**。

### 4.1 追踪止盈触发条件（Activation）

当满足其一时启用追踪止盈：
- 期权 `mid` 盈利 **≥ +15%**
- 或标的走出一段顺风（PUT：创新低；CALL：创新高）并维持结构（仍在 VWAP 方向一侧）

### 4.2 追踪参数（Trail）建议动态化

建议按当日波动分桶（与 T 分桶一致）：
- 高波动：回撤 **15%**（更宽，避免被噪声扫掉）
- 中波动：回撤 **12%**
- 低波动：回撤 **10%**（更紧，锁利润）

### 4.3 进一步锁盈（可选）

当浮盈变大时逐步收紧追踪：
- 浮盈 ≥ +30%：trail 下调 2-3pp
- 浮盈 ≥ +50%：trail 再下调 2-3pp

> 追踪止盈的核心是：让你“少做但稳”的策略吃到顺风段，同时避免从大赚回吐到亏。

## 5) 参数表（建议默认值）

| 参数 | 建议默认 | 说明 |
|---|---:|---|
| 0DTE 禁入窗口 | 09:30-10:00 | 禁止 0DTE 新开仓 |
| 0DTE 入场阈值 | 12 | PUT：≤ -12；CALL：≥ +12 |
| 连续确认 N | 2（跳变用 3） | 5 秒频率下约 10-15 秒 |
| 结构确认 | 2 根 1m 收盘在 VWAP 同侧 | 防 V 反转底部/顶部入场 |
| 结构失效止损 | 2 根 1m 收盘穿回 VWAP | 最高优先级退出 |
| 时间止损 T | 3 / 5 / 8 分钟 | 按 09:30-10:00 `rangePct` 分桶 |
| 最小顺风延续 | +5% mid 或 标的新高/新低 | T 内未满足则退出 |
| 0DTE PnL 兜底 | -25%（mid） | 过渡值，后续可动态化 |
| 硬安全阀 | -40% | 最后断尾 |
| 追踪止盈触发 | +15%（mid） | 达到后启用 TSLPPCT |
| 追踪回撤 | 10% / 12% / 15% | 低/中/高波动 |

## 6) 决策流程（实现时的检查顺序）

### 入场流程（每个标的）

1. 检查交易窗口：
   - 若在 09:30-10:00 且合约为 0DTE → **禁止开仓**（可改选 1DTE/2DTE）
2. score 达标：PUT `≤ -12` / CALL `≥ +12`
3. 连续确认 N 次达标
4. 结构确认：1m 收盘在 VWAP 同侧连续 2 根，且无强反转形态
5. 下单（LO/MO/ELO 按现有系统）

### 持仓退出流程（每个持仓，循环检查）

1. **结构失效止损**（Level A）
2. **时间止损**（Level B）
3. **PnL 兜底**（Level C）
4. 若达到追踪止盈触发条件 → 启用 `TSLPPCT` 并按波动分桶设置 trail

## 7) 验证方法（用你们现有分析方式即可）

建议对至少两天做反事实验证：
- **02-13（V 反转日）**：应显著减少/避免在底部附近开 0DTE，或更早因结构失效/时间止损退出，降低尾部亏损。
- **02-12（有效趋势日）**：不应因阈值过严而完全错过（因此本方案采用 `-12` 而不是固定 `-15`）。

关键评估指标：
- 每日交易次数（应下降）
- 最大单笔亏损/最大回撤（应下降）
- 盈亏比与期望值 EV（应改善）
- “被硬安全阀捕获”的次数（应显著下降）

## 8) 后续迭代方向（不阻塞 V1 落地）

- 将 `-25%` 兜底止损从固定值改为：按当日波动/点差/IV 状态动态化。
- 引入更可靠的反转识别（例如：VWAP 失守 + 放量 + 1m 结构组合），减少误杀。
- 若未来换券商/接口支持组合单，再评估“定义风险价差（Bear Put Spread）”以进一步稳定 EV。
