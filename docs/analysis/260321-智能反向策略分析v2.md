### 2.2 v1 核心发现的 Week 3 验证

#### 发现 1: 极端分数 = 均值回归信号?

v1 结论: marketScore < -35 时，BEAR(买 Put)胜率仅 17%，反向(买 Call)有效。

Week 3 验证:
- mktScore < -35 的 REV_BEAR 交易共 15 笔，胜率 47%，PnL -$394
- **部分验证**: 胜率确实高于 BEAR 在极端区的 17%，但 47% 仍不足以盈利
- **关键偏差**: v1 预测极端区均值回归"概率高"，Week 3 实测显示均值回归确实发生了(约一半交易盈利)，但**回归幅度不够覆盖止损笔的亏损**

v1 预测 vs 实测对比:

| v1 预测 | Week 3 实测 | 结论 |
|---------|-----------|------|
| 极端区均值回归概率 ~83% | 实测胜率 47% | **过于乐观** -- v1 的 83% 基于 6 笔小样本 |
| 反向操作 PnL 显著为正 | 实际 PnL -$394 | **未验证** -- 胜率不错但盈亏比太差 |
| 中等区域动量延续 | 3/17 BULL 在中等区 WR=38%, PnL=-$172 | **部分验证** -- 动量策略依然不稳定 |

**修正后的结论**: 极端分数(mktScore < -35)确实提高了均值回归的**概率**(从约 30% 提升到约 47%)，但均值回归的**幅度**不足以覆盖当市场继续下跌时的亏损。问题不在方向判断，**而在止盈止损参数**。

#### 发现 2: 分量分歧度是高价值信号?

| 时段 | 止盈% | 止损% | 追踪触发% | 追踪回撤% |
|------|-------|-------|----------|----------|
| EARLY (开盘) | 50% | 35% | 30% | 15% |
| MID (盘中) | 40% | 30% | 25% | 12% |
| LATE (尾盘前) | 30% | 25% | 20% | 10% |

- 用户配置 exitRules: takeProfitPercent=45%, stopLossPercent=35%
- MIT 保护单默认回退: 50% (双向)

### 3.3 止盈止损参数优化建议

基于 100 笔交易数据，提出以下参数调优方案:

#### 方案 A: 收紧止损线（P0 优先级）

**问题**: 当前 MID 阶段止损 30%、LATE 阶段 25%，对期权来说太宽。100 笔中约 40 笔以止损退出，平均亏损 -$69.5/笔，是总亏损的主要来源。

**建议调整**:

| 时段 | 当前止损% | 建议止损% | 理由 |
|------|----------|----------|------|
| EARLY | 35% | **25%** | 开盘波动剧烈，早发现错误早退出 |
| MID | 30% | **20%** | Week 3 大额亏损集中在 25-30% 区间 |
| LATE | 25% | **18%** | 0DTE 尾盘 Theta 急剧衰减，不应持有深度亏损仓位 |
| FINAL | 20% | **15%** | 收盘前应更激进地止损 |

**预期效果**: 假设 10 笔止损中有 6 笔在更紧的止损线被拦截(比当前少亏 5-10%)，可减少亏损约 $300-$500/周。代价是可能增加 2-3 笔"过早止损后反弹"的遗憾交易。

**回测验证方法**: 对 Week 3 的 10 笔止损交易，回溯其价格路径，计算在建议止损线下的实际退出价格和 PnL。

#### 方案 B: 降低止盈阈值 + 分阶段止盈（P1 优先级）

**问题**: 当前 EARLY 止盈 50%、MID 止盈 40%，对于 0DTE 期权来说目标过高。大多数 0DTE 交易的峰值盈利在 8%-25% 区间，能达到 40%+ 的极少。

**建议调整**:

| 时段 | 当前止盈% | 建议止盈% | 理由 |
|------|----------|----------|------|
| EARLY | 50% | **30%** | Week 3 数据显示大多数盈利在 15-25% |
| MID | 40% | **25%** | 中盘动量减弱，降低期望 |
| LATE | 30% | **20%** | 尾盘时间有限，15-20% 盈利即应锁定 |
| FINAL | 20% | **15%** | 维持或微调 |

**补充建议**: 考虑增加"部分止盈"机制 -- 当盈利达到 15% 时，平掉 50% 仓位锁利，剩余 50% 继续持有追求更高收益。这能解决"到了 15% 没卖，回落后只剩 4%"的问题。

#### 方案 C: 收紧 Trailing Stop 回撤阈值（P1 优先级）

**问题**: 当前 trailing stop 回撤阈值 10-15%，部分交易从 15%+ 峰值回落到 4% 才被触发。

**建议调整**:

| 时段 | 当前回撤% | 建议回撤% | 当前触发% | 建议触发% |
|------|----------|----------|----------|----------|
| EARLY | 15% | **12%** | 30% | **15%** |
| MID | 12% | **10%** | 25% | **12%** |
| LATE | 10% | **8%** | 20% | **10%** |
| FINAL | 8% | **6%** | 15% | **8%** |

**关键改善**: 降低触发点(从 30% 到 15% 等)意味着更早开始追踪，确保更多盈利交易被追踪止损保护。

**预估改善**: 以 #105 NVDA 为例:
- 当前: 峰值 15.3% -> 回撤 11.1% -> 退出 +4.2%
- 建议: 峰值 15.3% -> 回撤 10%(MID 阈值) -> 退出 ~+5.3%（改善约 $10）
- 如果触发点降至 12%: 12% 时即开始追踪，峰值 15.3% -> 回撤 10% -> 退出 ~+5.3%

以 #94 SPY 为例:
- 当前: 峰值 8.4% -> 回撤 14.4% -> 退出 -6.25%
- 如果触发点降至 8%: 8% 时开始追踪，峰值 8.4% -> 回撤 10% -> 退出 ~-1.6%（减亏 $28）
- 但注意: 8.4% 峰值时触发后如果用 8% 回撤，退出价约 0.4%，而非 -6.25%

#### 方案 D: 0DTE 专用参数（P0 优先级）

**问题**: 0DTE 期权的 Theta 衰减特性与非 0DTE 完全不同。当前系统虽有 0DTE 波动率调整，但基础参数仍沿用通用设置。

**建议**: 为 0DTE 期权建立独立参数组:

| 参数 | 通用 | 0DTE 专用 | 理由 |

### 4.3 shouldReverse() 条件优化建议

基于 Week 3 数据，对 v1 的 shouldReverse() 提出以下修改:

**修改 1: 将分歧度下限从 0.3 上调至 0.5**

理由: Week 3 数据显示分歧度 0.5-0.8 是最佳区间(WR 57%, +$28.4/笔)，而 0.3-0.5 区间表现平平(WR 50%, -$10.5/笔)。

**修改 2: 增加分歧度上限 1.0**

理由: 分歧度 > 0.8 时信号矛盾过于剧烈(WR 40%, -$43.8/笔)。#92 QQQ(分歧度 1.22)就是一个典型的"信号矛盾"案例(mkt 极端看跌但 intra 极端看涨)，不应触发反向。

**修改 3: 增加最大持仓时间限制**

理由: #95 NVDA 持仓 93 分钟才止损，在 0DTE 期权中是灾难性的。建议反向交易最大持仓不超过 45 分钟(0DTE)或 90 分钟(非 0DTE)。

**修改 4: 反向交易使用更紧的止损参数**

理由: 反向交易是对信号的"赌反"，不确定性更高，应该更快止损。建议反向交易止损线在正常策略基础上收紧 30%(如正常 25% -> 反向 18%)。

**修改 5: 增加 SPY 特别风控**

理由: Week 3 中 SPY 的反向交易表现极差:
- #94: -$60
- #99: -$48
- #101: -$18
- #104: -$240
共 4 笔全部亏损，总计 -$366。SPY 作为大盘 ETF，在 marketScore 极端时可能正在经历真正的趋势性下跌(而非超卖反弹)，反向策略风险更大。

建议: SPY/QQQ 等大盘 ETF 的反向门槛应高于个股(如分歧度要求 0.7 而非 0.5)。

**优化后的 shouldReverse() 伪代码**:

```typescript
function shouldReverseV2(
  marketScore: number,
  intradayScore: number,
  symbol: string,
  is0DTE: boolean,
): { reverse: boolean; confidence: 'HIGH' | 'MEDIUM' | 'LOW'; reason: string } {

  const normMarket = marketScore / 40;
  const normIntraday = intradayScore / 20;
  const divergence = Math.abs(normMarket - normIntraday);

  const marketExtreme = Math.abs(marketScore) > 35;
  const intradayExtreme = intradayScore < -14 || intradayScore > 15;

  // 大盘ETF需要更高的分歧度门槛
  const isIndexETF = ['SPY', 'QQQ', 'IWM'].includes(symbol);
  const divergenceMin = isIndexETF ? 0.7 : 0.5;
  const divergenceMax = 1.0;

  // 高置信反向: mkt 极端 + intra 不极端 + 分歧度适中
  if (marketExtreme && !intradayExtreme
      && divergence >= divergenceMin && divergence <= divergenceMax) {
    return {
      reverse: true,
      confidence: 'HIGH',
      reason: `大盘极端(${marketScore.toFixed(1)})日内企稳(${intradayScore.toFixed(1)})分歧度=${divergence.toFixed(2)}`
    };
  }

  // 中等置信反向: 双极端同向（但需更高分歧度门槛来过滤）
  if (marketExtreme && intradayExtreme) {
    const sameDirection =
      (marketScore < 0 && intradayScore < 0) ||
      (marketScore > 0 && intradayScore > 0);
    if (sameDirection && !isIndexETF) {
      return {
        reverse: true,
        confidence: 'MEDIUM',
        reason: `双极端同向(mkt=${marketScore.toFixed(1)},intra=${intradayScore.toFixed(1)})`
      };
    }
  }

  // 默认: 不反向
  return { reverse: false, confidence: 'HIGH', reason: '顺势操作' };
}
```

### 4.4 v1 vs v2 shouldReverse() 差异总结

| 维度 | v1 版 | v2 版(本文建议) | 变更理由 |
|------|------|---------|---------|
| 分歧度下限 | 0.3 | **0.5**(个股) / **0.7**(指数ETF) | 0.3-0.5 区间表现不佳 |

### 核心结论

1. **止盈止损参数是当前系统最大的可优化项**。100 笔交易中，止损交易的笔均亏损(-$93)远超追踪止损的笔均盈利(+$23)。收紧止损线(从 25-35% 降至 18-25%)是投入产出比最高的改进。

3. **分歧度阈值需从 0.3 上调至 0.5**。Week 3 数据明确显示 0.5-0.8 是最佳分歧度区间，0.3-0.5 的表现不如预期。同时应增加 1.0 上限以过滤信号矛盾的极端情况。

5. **系统的真正 alpha 来源是追踪止损风控，而非方向判断**。方向判断胜率 37% 为负贡献，追踪止损笔均 +$23 为正贡献。优化重点应放在风控参数上，而非追求更准确的方向预测。

### 第一性原理回顾

**当务之急不是让方向更准，而是让风控更紧。**

#### 10.3.1 时间维度：越晚入场越危险

3/19 同样验证了这一规律：
- 15:34 入场 NVDA #93: +$235（方向对 + 充足时间让利润兑现）
- 16:31 入场 NVDA #95: -$280（方向错 + 无时间容错 + Theta 急剧衰减）

**第一性原理解释**: 0DTE 期权在盘中最后 1-2 小时的 Theta 衰减是非线性加速的。假设一张 0DTE 期权的隐含波动率为 30%，在收盘前 1 小时其 Theta 衰减速率约为 3 小时前的 3-4 倍。这意味着：
- 14:00 入场：价格需要移动 X% 来覆盖 Theta 成本
- 16:00 入场：价格需要移动 3-4X% 来覆盖同样的 Theta 成本
- 容错空间随时间**非线性收缩**

这不是"方向"问题，也不是"止损"问题。这是一个 **物理约束**——入场越晚，即使方向正确，Theta 侵蚀利润的速度也越快，留给方向兑现的时间窗口越短。

| 维度 | #93 (赚 $235) | #95 (亏 $280) | 差异 |
|------|--------------|--------------|------|
| 入场时间 | 15:34 | 16:31 | 晚 57 分钟 |
| marketScore | -39.57 | -38.92 | 几乎相同 |
| intradayScore | **-2.84** | **+0.36** | 关键差异 |
| 分歧度 | 0.85 | 0.99 | 相近 |
| 入场后价格路径 | 上涨达 24.1% | 持续下跌至 -29.5% | 完全相反 |

- intraScore = -2.84：日内仍在下跌，但跌幅收窄（大盘 -39 但日内只 -2.84），说明**日内有企稳迹象，均值回归正在发生**
- intraScore = +0.36：日内已经转正（微涨），意味着**日内反弹可能已经完成**，再买入 Call 实际上是在追涨日内反弹的尾巴

第一性原理推导：智能反向的逻辑是"大盘极端超卖 + 日内企稳 = 均值回归"。当 intraScore 从负转正，意味着均值回归**已经发生过了**。此时入场不是在赌"回归将要发生"，而是在赌"回归还能继续"——这是一个概率显著更低的赌注。

**诊断结论**: #95 的失败不是"方向错了"，而是 **"入场太晚，错过了均值回归的主要窗口"**。intraScore 从负转正是一个关键的时机信号——它标志着均值回归动能的耗竭。

### 10.4 用户观点验证：QQQ #106 止损后是否反弹？

#### 事实还原

QQQ #106 的时间线：

- 16:52 触发 0DTE 兜底止损 $1.40（-25.7%, -$247）

#### 是否反弹的分析

无法从现有日志直接获取止损后的 QQQ 价格路径。但可以从以下逻辑推断：

1. **0DTE 期权在收盘前 15 分钟的特性**: 即使 QQQ 标的价格（SPY/QQQ 正股）出现小幅反弹，0DTE 期权的 Theta 衰减在最后 15 分钟极其剧烈。一张 ATM 0DTE 期权在收盘前 15 分钟，时间价值几乎为零，只剩内在价值。要从 $1.40 回升到入场价 $1.89，需要标的价格大幅变动（约 $0.49 的内在价值增量，对应 QQQ 正股约 0.5-1.0 美元的移动，取决于 Delta）。

2. **即使正股反弹，期权未必跟涨**: 在 0DTE 最后 15 分钟，期权 Delta 接近 0 或 1（二元化），Vega 接近零。这意味着即使 QQQ 正股反弹 0.3%，如果期权不在价内足够深，价格基本不变。

3. **但用户的直觉可能在更宏观的层面是对的**: 如果入场时间更早（比如 15:00 而非 16:43），那么 QQQ 的日内均值回归有更充裕的时间兑现，即使中间有 -15% 的回撤，后续可能确实反弹。**问题不是 #106 这一笔是否该更宽止损，而是 #106 这一笔根本就不应该发生**——16:43 入场 0DTE 本身就是一个时机错误。

#### 结论

对于 #106 这笔具体交易：止损后即使有反弹，期权价格在 0DTE 最后 15 分钟几乎无法受益。加宽止损不会改善这笔交易的结果，反而可能让亏损从 -$247 扩大到 -$350+（持有至到期归零）。

但用户直觉的底层逻辑——"方向对的时候不应该被过早洗出"——是正确的。只是这个逻辑应该用来改善 **入场时机**（不在 16:30 后入场 0DTE），而不是用来放宽止损。

> 注: 锁利不足的 7 笔中，3 笔最终为正（+$76, +$20, +$40 合计+$136），4 笔最终为负（-$15, +$1, -$14, -$8(#96 按时机归因)... 修正: #97 +$76, #98 +$74, #100 +$20, #101 -$15, #102 +$1, #103 -$14, #105 +$40）。锁利不足组最终 PnL = +$76+$74+$20-$15+$1-$14+$40 = +$182。但如果按峰值的 70% 计算理论可锁利润：(19.2%+20%+16.9%+11.6%+11.7%+14.2%+15.3%) x 70% x 对应手数金额 ≈ 显著更高。

**修正后的归因汇总（按 PnL 影响）**:

| 归因类别 | 笔数 | 实际PnL | 理论可改善PnL | 说明 |
|---------|------|---------|-------------|------|
| 成功 | 2 | +$311 | -- | 已是理想结果 |
| 锁利不足 | 7 | +$182 | +$500~$600(估) | 如果在峰值 70% 处退出 |
| 时机错误 | 3 | -$113 | $0(不入场) | 这些交易应被过滤掉 |
| 方向错误 | 3 | -$767 | -$200~-$300(紧止损) | 方向错 + 止损宽叠加 |

**关键洞察**:

1. **方向错误仅占 20%（3/15 笔），但贡献了 83% 的亏损（-$767）**。这 3 笔（#95 -$280, #104 -$240, #106 -$247）有两个共同特征：入场时间晚（16:19, 16:31, 16:43）且止损幅度大（-25%~-30%）。

2. **锁利不足占 47%（7/15 笔），这些交易方向全部正确但仅贡献 +$182，远低于理论峰值利润**。如果锁利机制更激进（在峰值回撤 5-8% 而非 10-15% 时退出），估计可多锁 $300-$400 利润。

3. **方向错误和时机错误的叠加才是灾难的根源**。纯粹的方向错误（市场继续跌）如果搭配合理的入场时间过滤和止损，亏损可控。16:00 后禁止入场 0DTE 这一条规则，就能直接消除 -$767 中的 -$487（#104 + #106）。

### 10.6 对 v2 报告结论的修正

#### 对止损参数建议的修正

v2 报告方案 A 建议全面收紧止损线。修正后的建议更加细分：

| 场景 | v2 建议 | 修正建议 | 理由 |
|------|---------|---------|------|
| 方向正确+早盘入场 | 收紧止损 | **维持或适度放宽**，重点在锁利 | 80% 的交易方向是对的，过紧止损会增加"被洗出后反弹"的次数 |
| 方向正确+晚盘入场(16:00后) | 收紧止损 | **禁止入场** | 根源问题是时机不是止损 |
| 方向错误(从未转正) | 收紧止损 | **同意收紧**，但更重要的是入场过滤 | 这些交易的 intraScore 通常已转正（均值回归窗口已过） |

**综合建议**:

1. **P0 — 0DTE 入场时间窗口限制**
   - 0DTE 期权禁止在 16:00（东部）后新开仓
   - 预期效果：直接消除 #104(-$240) 和 #106(-$247) 类交易，减亏 ~$500/周
   - 成本：损失 16:00 后可能的盈利交易（从数据看该时段盈亏比极差，净效果为正）

2. **P0 — intraScore 入场过滤**
   - 当 intraScore 从负转正（> 0）时，不再触发反向入场（均值回归窗口已过）
   - 预期效果：过滤掉 #95 类交易（intra=+0.36 时入场，从未转正），减亏 ~$280
   - 数据支撑：3/19 两笔 NVDA 对比——intra=-2.84 赚 $235，intra=+0.36 亏 $280

3. **P1 — 锁利机制强化（优先级高于收紧止损）**
   - 降低追踪止损触发点至 8-10%（当前 15-25%）
   - 当峰值达 10% 后追踪回撤收紧至 5-6%（当前 10-15%）
   - 预期效果：7 笔"锁利不足"交易估计可多保留 $300-$400 利润
   - 回测验证：取 #100 NVDA 为例——峰值 16.9%，如果 10% 触发追踪 + 6% 回撤，退出点约 10.9%（对比当前 2.9%），多锁 ~$80

4. **P2 — 止损参数选择性收紧（而非全面收紧）**
   - 仅对 intraScore > 0（均值回归已过）的交易收紧止损至 15%
   - 对 intraScore < -5（均值回归进行中）的交易维持当前止损 25-30%
   - 理由：方向正确时（80%概率），需要给价格波动留出空间；方向可能错误时（intra 已转正），早止损减少损失

### 10.7 修正后的优化路线图

```
v2 原路线:
第 1 步: 止损参数收紧 → 第 2 步: 追踪止损调优 → 第 3 步: 止盈调优 → 第 4-5 步: 反向策略

修正后路线:
第 1 步（本周）: 0DTE 入场时间窗口限制（16:00 后禁止新开 0DTE）
              + intraScore 入场过滤（intra > 0 时禁止反向入场）
              → 预期减亏: $500-$800/周

第 2 步（下周）: 锁利机制强化（追踪触发点 10% → 8%, 追踪回撤 12% → 6%）
              → 预期增收: $300-$400/周

第 3 步（第 3 周）: 选择性止损收紧（仅针对 intra > 0 的交易）
              → 预期减亏: $100-$200/周

第 4 步（第 4 周）: 综合评估三步效果，微调参数
```

**关键区别**: v2 路线以"收紧止损"为首步，修正后路线以"入场时机过滤"为首步。原因是入场过滤的预期收益更高（$500-$800 vs $250-$400），且风险更低（过滤不会影响已入场的好交易，而收紧止损可能洗出方向正确的交易）。

### 10.8 本章结论

v2 报告的核心方向——"优化风控参数比追求方向准确更重要"——仍然正确。但 **"风控"的内涵需要重新定义**：

- 风控 ≠ 仅仅是止损参数
- 风控 = **入场时机过滤（什么时候不该进场）+ 锁利机制（方向对了如何变现）+ 止损参数（方向错了如何控制损失）**

从 3/19-3/20 的 15 笔实盘数据看：
- 80% 的交易方向是对的（峰值 PnL > 0）
- 但只有 47% 最终盈利
- 33 个百分点的差值来自"锁利不足"和"时机错误"
- 大额亏损（-$767）集中在 20% 的方向错误交易中，且这些交易的共同特征是**入场时间晚 + 止损宽**

最精确的诊断是：**方向判断基本正确，但入场时机过滤不足 + 锁利机制过于宽松 + 方向错误时止损未能及时控制损失，三者叠加导致好方向转化为差结果。**

优化应按"投入产出比"排序：入场时机过滤 > 锁利机制强化 > 选择性止损收紧。
