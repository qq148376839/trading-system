# 资金分配方式分析：按组平分 vs 按标的平分

> 创建时间: 2026-03-13

## 问题背景

当前资金分配逻辑在 `capital-manager.service.ts` 中：

```
maxPerSymbol = min(allocatedAmount / denominator, allocatedAmount * concentration)
```

### 问题 1: BY_SYMBOL 模式资金过于分散

- 总资金 $2000，7 个标的 → 每标的 $285.7 → 只能买 1 张合约
- `concentration = 0.33`（默认）→ `min(285.7, 660) = $285.7`
- 对于期权策略，$285 的预算严重限制了合约选择空间

### 问题 2: BY_GROUP 模式被 concentration 封顶

- 2 组时：`min(2000/2, 2000*0.33) = min(1000, 660) = $660`
- 同组互斥已保证每组最多 1 个持仓，0.33 的集中度限制不合理
- 天然集中度上限应为 `1/groupCount`（2 组 = 0.5）

### 问题 3: 前端无 maxConcentration 配置入口

- 用户无法手动调整集中度参数

## 解决方案

### 新增 `capitalSplitMode` 配置

存储位置：`strategies.config.correlationGroups.capitalSplitMode`

| 模式 | 分母 | 集中度 | 示例（$2000, 2组, 7标的）|
|------|------|--------|------------------------|
| BY_SYMBOL（默认）| symbolPool.length (7) | 0.33 | min(285.7, 660) = $285.7 |
| BY_GROUP | groupCount (2) | 1/groupCount (0.5) | min(1000, 1000) = $1000 |

### 优先级规则

`resolveDenominator()` 返回 `{ denominator, concentration }`：

1. **survivorCount > 0**（Phase D 竞价）→ `{ survivorCount, maxConcentration ?? 0.33 }`
2. **capitalSplitMode === 'BY_GROUP'** → `{ groupCount, 1/groupCount }`
3. **默认 BY_SYMBOL** → `{ symbolPool.length, 0.33 }`

### 为什么 BY_GROUP 模式下 concentration = 1/groupCount

- 同组互斥机制保证同组内最多持有 1 个标的
- 因此实际最大持仓数 = 组数
- 集中度上限 = 1/组数 是数学上的天然上限
- 无需额外的 maxConcentration 保护

## 影响范围

### 需要修改的文件

1. `api/src/services/capital-manager.service.ts` — 新增 `resolveDenominator`，修改 `getMaxPositionPerSymbol` 和 `requestAllocation`
2. `api/src/routes/quant.ts` — PUT correlation-groups 增加 capitalSplitMode 校验
3. `frontend/app/quant/strategies/[id]/page.tsx` — 新增 Select 组件
4. `api/src/routes/quant.ts` — GET strategy 增加 allocationValue/allocationtype 字段

### 不需要修改的文件

- `option-intraday-strategy.ts` / `schwartz-option-strategy.ts`：调用 `getMaxPositionPerSymbol(strategyId)` 不传 survivorCount → 自动走 BY_GROUP 分支
- `strategy-scheduler.service.ts`：Phase D 传 survivorCount → 走最高优先级分支

### 数据库

- 无 DDL 变更，`capitalSplitMode` 存于现有 `strategies.config` JSONB 字段内
