## 3. 阶梯锁利问题诊断

### 当前配置

```
profitLockSteps: [
  { threshold: 6,  floor: 2 },   // 回吐空间 4% (67% of peak)
  { threshold: 12, floor: 6 },   // 回吐空间 6% (50% of peak)
  { threshold: 18, floor: 12 },  // 回吐空间 6% (33% of peak)
  { threshold: 24, floor: 18 },  // 回吐空间 6% (25% of peak)
  { threshold: 30, floor: 26 },  // 回吐空间 4% (13% of peak)
]
```

### 问题根因

1. **低台阶回吐比例畸形**: 6% 台阶允许回吐 67%，12% 台阶允许回吐 50%。这意味着一个赚到 12% 的交易，允许跌回 6% 才触发——回吐了整整 6 个百分点
2. **底线与台阶之间的 gap 不对称**: 低台阶 gap 占比太大，高台阶 gap 占比合理。30% 台阶只允许回吐 13%，但 12% 台阶允许回吐 50%

### 实际发生的情况

12% 台阶的 18 笔交易中：
- 峰值平均 14.9%，但实际退出平均仅 4.2%
- 最差案例: TSLA 3/30 峰值 14.4% → 退出仅 1.8%（**回吐 88%**）
- QQQ 3/30 峰值 12.4% → 退出仅 1.0%（**回吐 92%**）
- 根本原因: 期权在 0DTE 下价格波动极快，从 12-15% 利润回落到 6% 底线经常只需要几分钟，而这几分钟还在继续下跌到 1-5%

## 4. 参数建议

### 方案A（推荐）：收紧底线 + 加密台阶

```
profitLockSteps: [
  { threshold: 6,  floor: 3 },   // 回吐 3% (50%) — 原 4%
  { threshold: 10, floor: 6 },   // 新增台阶，覆盖 10-12% 区间
  { threshold: 15, floor: 10 },  // 回吐 5% (33%) — 原 12%→6% gap=6%
  { threshold: 20, floor: 15 },  // 回吐 5% (25%)
  { threshold: 25, floor: 20 },  // 回吐 5% (20%)
  { threshold: 30, floor: 26 },  // 保持不变
]
```

**数据支撑**: 历史回测模拟，仅收紧底线（方案A简化版 12%→8%）就能在 25 笔锁利交易上改善 +81.5% 的总 PnL。

### 方案B（保守）：仅调整前两级

```
profitLockSteps: [
  { threshold: 6,  floor: 3 },   // 回吐 3% — 原 4%
  { threshold: 12, floor: 8 },   // 回吐 4% — 原 6%（核心改动）
  { threshold: 18, floor: 13 },  // 回吐 5% — 原 6%
  { threshold: 24, floor: 19 },  // 回吐 5% — 原 6%
  { threshold: 30, floor: 26 },  // 不变
]
```

### 方案比较

## 5. 其他参数建议

### 止损 stopLossPercent: 35 → 建议 30

**数据支撑**:
- 当前 20 笔止损平均亏损 -28.9%
- 止损分布: 25-27% 有 10 笔(50%)，27-30% 有 3 笔，30-35% 有 5 笔，35%+ 有 2 笔
- 35% 止损很少被用到（实际 0DTE 兜底止损在 25-30% 就已触发）
- 降到 30% 可以减少极端亏损（-34.2%, -35.4%, -41.5% 这几笔）

### zdteCooldownMinutes: 15 → 建议 20-25

**数据支撑**:
- 3/30 当天 TSLA 连续 4 笔入场，前 3 笔都是锁利退出后快速重入
- 15 分钟冷却不足以让市场噪声消退
- 0DTE 期权对时间极度敏感，过于频繁入场增加费用拖累

### noNewEntryBeforeCloseMinutes: 180 → 合理

- 3 小时前停止入场对 0DTE 是合理的
- 不建议调整
