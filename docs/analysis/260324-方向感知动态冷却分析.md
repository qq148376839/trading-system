# 方向感知动态冷却（Direction-Aware Dynamic Cooling）

**日期**: 2026-03-24
**关联文档**: `260313-组内竞价轮换与冷却升级分析.md`, `260321-智能反向策略分析v2.md`

---

## 一、问题回顾

当前组内冷却是固定时间（10 分钟 flat），不区分方向、盈亏、市场状态。这不符合第一性原理——冷却的本质是**防止在相同市场状态下重复同一个错误**，如果市场状态已经变了（方向翻转），冷却就失去了意义。

**用户提出的核心思想**：方向变化 = 市场状态变化 = 不需要冷却。

---

## 二、第一性原理推导

### 2.1 冷却的本质是什么？

冷却不是惩罚，是一个**信息等待窗口**。

上一笔交易退出时，市场给了你一个信号（盈利或亏损）。冷却期的目的是等待**足够的新信息**进入市场，使得下一笔交易不再是对上一笔的简单重复。

**核心公式**：

```
需要冷却的程度 ∝ 当前市场状态与上次入场时的相似度
```

- 相似度高 → 需要更多冷却（同一个"坑"，不要再跳）
- 相似度低 → 冷却可以缩短或取消（新的局面，新的机会）

### 2.2 "市场状态相似度"由什么决定？

从交易系统可获取的信息中，衡量市场状态变化的维度有 5 个，按信息量从大到小排列：

| 维度 | 信息含量 | 说明 |
|------|---------|------|
| **方向翻转** | 最高 | call→put 或 put→call，代表市场结构性变化 |
| **分数变化幅度** | 高 | finalScore 变化越大，市场状态越不同 |
| **时间流逝** | 中 | 时间本身带来新信息（新K线、新成交量） |
| **上笔盈亏** | 中 | 市场对上一个方向的"反馈" |
| **连续同向次数** | 低但重要 | 重复同方向 = 可能在和趋势较劲 |

**为什么方向翻转的信息量最大？**

方向翻转意味着评分系统的输入（marketScore、intraScore）发生了质变而非量变。在智能反向框架下，方向翻转可能来自：
1. marketScore 从极端恢复到中等（均值回归结束 → 动量恢复）
2. intraScore 从负转正（日内方向翻转）
3. 分歧度变化导致 smartReverse 判定翻转

每一种都代表市场微观结构的实质性变化。

### 2.3 为什么固定时间冷却违反第一性原理？

固定 10 分钟冷却隐含一个假设：**市场状态需要恰好 10 分钟才能充分变化**。

但现实是：
- 在高波动日（如 3/19 marketScore=-39），市场状态可能 2 分钟就彻底翻转
- 在低波动日，10 分钟内市场可能纹丝不动
- 方向已经翻转时，再等 8 分钟毫无意义
- 方向没变但分数微调时，10 分钟可能不够

**正确的冷却应该是状态驱动的，不是时间驱动的。**

## 三、动态冷却模型设计

### 3.1 核心机制：重入准备度评分（Reentry Readiness Score）

不使用固定时间冷却，改为计算一个连续的**重入准备度分数**（0~1），当分数超过阈值时允许重入。

```
reentryReadiness = Σ(weight_i × factor_i)

if reentryReadiness >= threshold → 允许参与竞价
if reentryReadiness < threshold → 跳过，选同组第二名
```

### 3.2 五因子模型

#### Factor 1: 方向变化（权重 0.35）

```typescript
directionFactor = (currentDirection !== lastExitDirection) ? 1.0 : 0.0
```

- 最重要的单一因子
- 二值信号：变了就是变了
- `lastExitDirection` 存储在扩展后的 `recentExits` 中
- `currentDirection` 来自候选人的 `intent.metadata.optionDirection`

#### Factor 2: 分数变化幅度（权重 0.25）

```typescript
scoreDelta = |currentFinalScore - lastExitFinalScore|
scoreFactor = min(1.0, scoreDelta / scoreChangeThreshold)
```

- 连续信号：即使方向没变，分数大幅变化也代表市场状态不同
- 例：上次 finalScore=-9.5（刚过阈值勉强入场），这次 finalScore=-15（强信号）
- `scoreChangeThreshold` 的取值：从数据看，finalScore 范围约 [-20, +20]，变化 5 分以上代表显著变化

**scoreChangeThreshold 动态化**：
```typescript
// 用近期分数波动的标准差来归一化
// 波动大的日子，5分变化不算什么；波动小的日子，3分就很显著
scoreChangeThreshold = max(3, recentScoreStdDev * 1.5)
```

如果无法实时计算标准差，用固定值 5.0 作为起点，后续可调。

#### Factor 3: 时间衰减（权重 0.20）

```typescript
minutesSinceExit = (now - lastExitTime) / 60000
timeFactor = min(1.0, minutesSinceExit / baseCooldownMinutes)
```

- 时间本身带来新信息，即使其他维度没变
- `baseCooldownMinutes` 根据波动率动态调整：

```typescript
// 高波动 = 市场变化快 = 冷却窗口短
// 低波动 = 市场变化慢 = 冷却窗口长
if (rangePct >= 0.65) baseCooldownMinutes = 5    // 高波动
else if (rangePct >= 0.45) baseCooldownMinutes = 8  // 中波动
else baseCooldownMinutes = 12                       // 低波动

// 0DTE vs 非0DTE 修正
if (!is0DTE) baseCooldownMinutes *= 1.5  // 非0DTE 节奏更慢
```

#### Factor 4: 上笔盈亏与方向的交互（权重 0.12）

这是最微妙的因子。盈亏的含义取决于方向是否变化：

```typescript
if (directionChanged) {
  // 方向变了
  // 上笔亏损 + 方向变化 = 市场否定了旧方向，新方向可能对 → 高准备度
  // 上笔盈利 + 方向变化 = 旧方向还在赚钱，为什么要换？→ 中低准备度
  outcomeFactor = (lastPnL <= 0) ? 0.9 : 0.4
} else {
  // 方向没变
  // 上笔盈利 + 同方向 = 动量延续信号，可以再试 → 中高准备度
  // 上笔亏损 + 同方向 = 市场刚否定了这个方向 → 低准备度
  outcomeFactor = (lastPnL > 0) ? 0.7 : 0.1
}
```

**第一性原理解释**：
- 亏损是市场的反馈："这个方向错了"。如果你换了方向，这个反馈是**正向信号**（你听到了市场的声音）
- 盈利是市场的反馈："这个方向对了"。如果你换了方向，这个反馈是**警告信号**（你在逆转一个有效趋势）
- 但注意：盈利退出 + 同方向重入 = 动量交易，有合理性

#### Factor 5: 连续同向交易次数（权重 0.08）

```typescript
// 连续同方向交易越多，越可能在和市场较劲
consecutiveFactor = max(0, 1.0 - 0.35 * consecutiveSameDirectionCount)
// 0次: 1.0  |  1次: 0.65  |  2次: 0.30  |  3次: 0.0（有效阻止）
```

这是一个自动升级的保护：
- 第 1 次同方向：微小惩罚
- 第 2 次：明显惩罚
- 第 3 次：几乎完全阻止

### 3.3 阈值设计

```typescript
reentryThreshold = 0.50  // 基础阈值，可配置
```

**为什么是 0.5？** 因为在 5 因子模型中：
- 纯方向翻转（其他因子全 0）：0.35 × 1.0 = 0.35 → 不够（需要至少一个辅助因子）
- 方向翻转 + 3分钟过去（timeFactor≈0.4）：0.35 + 0.20×0.4 = 0.43 → 还不够
- 方向翻转 + 3分钟 + 上笔亏损：0.35 + 0.08 + 0.12×0.9 = 0.54 → **通过**
- 同方向 + 10分钟过去 + 上笔盈利：0 + 0.20 + 0.12×0.7 = 0.28 → 不通过
- 同方向 + 10分钟 + 盈利 + 分数大变(Δ=8)：0 + 0.25 + 0.20 + 0.08 = 0.53 → **勉强通过**

这意味着：
1. **方向翻转是必要非充分条件**——翻转了还需要一点时间或盈亏信号确认
2. **同方向几乎不可能快速重入**——除非分数变化极大 + 时间足够 + 上笔盈利
3. **最快重入路径**：方向翻转 + 上笔亏损 → 约 2-3 分钟即可重入

这个 0.5 阈值可以根据实战调整。提高阈值 = 更保守，降低 = 更激进。

## 四、场景模拟：3/19-3/20 数据回测

### 4.1 场景 A：3/20 全天 REV_BEAR（同方向持续）

当天 marketScore 持续 -39，所有竞价结果都是 CALL（反向做多）。

| 交易 | 方向 | 上笔结果 | timeFactor | 准备度 | 结果 |
|------|------|---------|-----------|--------|------|
| #96 TSLA (首笔) | CALL | 无 | N/A | 1.0 | **入场** → -$8 |
| #97 QQQ | CALL(同) | -$8(亏) | ~0.3 | 0+0.06+0.06+0.01+0.065=**0.195** | **跳过** |
| #98 NVDA | CALL(同) | -$8 | ~0.5 | 0+0.10+0.06+0.01+0.065=**0.235** | **跳过** |
| (...后续同向全部被阻止，直到方向翻转或足够时间过去) | | | | | |

**效果**：在同向持续日，每组仅允许首笔交易。#104(-$240) 和 #106(-$247) 被自然过滤。
**减亏预估**：避免 #99(-$48), #101(-$15), #104(-$240), #106(-$247) = **减亏 $550+**

但注意也过滤了 #97(+$76), #98(+$74), #100(+$20), #102(+$1), #105(+$40) = **损失盈利 $211**

**净效果**：减亏约 **+$340**

### 4.2 场景 B：方向翻转（假设 score 从 BEAR 翻为 BULL）

假设某标的上次以 CALL 退出（亏损 -$50），下一轮竞价 finalScore 转正，方向变为 PUT：

```
direction: 1.0 × 0.35 = 0.35
scoreDelta(Δ=12): 1.0 × 0.25 = 0.25
time(2min/8min): 0.25 × 0.20 = 0.05
outcome(亏损+方向变): 0.9 × 0.12 = 0.108
consecutive(0): 1.0 × 0.08 = 0.08
Total: 0.838 → 远超阈值，立即允许
```

**效果**：方向翻转时几乎无延迟重入。

### 4.3 场景 C：同方向但分数大幅变化

上次 finalScore=-9.5（刚过阈值），亏损退出。15 分钟后新竞价 finalScore=-16（极强信号），方向仍为 CALL：

```
direction: 0.0 × 0.35 = 0.00
scoreDelta(Δ=6.5): 1.0 × 0.25 = 0.25
time(15min/8min): 1.0 × 0.20 = 0.20
outcome(亏损+同向): 0.1 × 0.12 = 0.012
consecutive(1): 0.65 × 0.08 = 0.052
Total: 0.514 → 勉强通过
```

**效果**：同方向但信号大幅增强 + 足够时间 → 允许。这是合理的——信号从"勉强"变为"极强"代表市场给了新信息。

### 4.4 场景 D：同方向、分数差不多、刚退出

上次 finalScore=-11，亏损退出。3 分钟后新竞价 finalScore=-12，方向仍为 CALL：

```
direction: 0.0 × 0.35 = 0.00
scoreDelta(Δ=1): 0.2 × 0.25 = 0.05
time(3min/8min): 0.375 × 0.20 = 0.075
outcome(亏损+同向): 0.1 × 0.12 = 0.012
consecutive(1): 0.65 × 0.08 = 0.052
Total: 0.189 → 远低于阈值，阻止
```

**效果**：这就是"重复同一个错误"——被正确阻止。

## 五、竞价流程集成设计

### 5.1 修改点：Phase C 竞价逻辑

当前 Phase C Phase 1 只保留每组 |finalScore| 最高的候选人。

修改后：

```
Phase 1（组内排名）:
  1. 所有候选人按 |finalScore| 降序排列
  2. 对每组的最高分候选人，计算 reentryReadiness
  3. 如果 readiness >= threshold → 选为组代表
  4. 如果 readiness < threshold → 标记为"方向冷却"，尝试同组第二名
  5. 第二名同样检查 readiness（第二名可能是不同标的、不同方向）
  6. 如果同组所有候选人都不通过 → 本轮该组不参与
```

### 5.2 需要扩展的数据结构

`recentExits` 从 `Map<string, number>` 扩展为：

```typescript
interface GroupExitRecord {
  cooldownEnd: number;          // 原有：冷却结束时间
  exitDirection: 'CALL' | 'PUT'; // 新增：退出时的方向
  exitFinalScore: number;       // 新增：退出时的 finalScore
  exitPnL: number;              // 新增：退出时的 PnL（金额或百分比）
  exitTime: number;             // 新增：退出时间戳（精确，用于 timeFactor）
  consecutiveSameDirection: number; // 新增：该组连续同方向次数
}

recentExits: Map<string, GroupExitRecord>
```

### 5.3 读取 recentExits 的位置变化

**当前**：Phase B `evaluateIdleSymbol()` 中检查 `recentExits`，如果同组有冷却中的标的，直接 return null。

**修改后**：Phase B 不再用 `recentExits` 阻止候选人进入竞价。改为在 Phase C 中用 `reentryReadiness` 评分来决定是否跳过。

理由：Phase B 应该负责信号质量评估，Phase C 负责竞争和风控过滤。方向感知冷却是一种竞争规则，属于 Phase C。

### 5.4 recordSymbolExit() 扩展

退出时记录更多信息：

```typescript
recordSymbolExit(strategyId, symbol, cooldownUntilStr, exitInfo: {
  direction: 'CALL' | 'PUT',
  finalScore: number,
  pnl: number,
}) {
  const group = getCorrelationGroup(symbol, correlationMap);
  const prev = crossState.recentExits.get(/* group key */);

  const consecutiveSameDir = (prev && prev.exitDirection === exitInfo.direction)
    ? prev.consecutiveSameDirection + 1
    : 0;

  crossState.recentExits.set(symbol, {
    cooldownEnd: new Date(cooldownUntilStr).getTime(),
    exitDirection: exitInfo.direction,
    exitFinalScore: exitInfo.finalScore,
    exitPnL: exitInfo.pnl,
    exitTime: Date.now(),
    consecutiveSameDirection: consecutiveSameDir,
  });
}
```

## 六、参数动态化策略

### 6.1 哪些参数应该是动态的？

| 参数 | 静态值 | 动态化方案 | 驱动因子 |
|------|--------|-----------|---------|
| `reentryThreshold` | 0.50 | 根据当日策略表现调整 | 当日 PnL |
| `baseCooldownMinutes` | 8 | 根据波动率调整 | rangePct |
| `scoreChangeThreshold` | 5.0 | 根据近期分数波动调整 | 分数标准差 |
| 因子权重 | 见 3.2 | **不动态化**（稳定性优先） | — |

### 6.2 reentryThreshold 动态化

```typescript
// 当日亏损越多 → 阈值越高 → 越难重入（自我保护）
// 当日盈利 → 阈值维持基准（不过度激进）
const dailyPnL = cancelCtx.dailyRealizedPnL ?? 0;
const baseDailyLossThreshold = -300; // 可配置

if (dailyPnL < baseDailyLossThreshold) {
  // 亏损超过阈值，提高重入门槛
  reentryThreshold = 0.50 + 0.15 * min(1.0, |dailyPnL| / (|baseDailyLossThreshold| * 2));
  // 亏$300: 0.575 | 亏$600: 0.65（上限）
} else {
  reentryThreshold = 0.50; // 基准
}
```

**第一性原理**：当日已经亏了不少，说明今天的市场对策略不友好。提高重入门槛 = 减少交易频率 = 控制总损失。这是一种自适应风控。

### 6.3 baseCooldownMinutes 动态化（timeFactor 的分母）

```typescript
// 波动率驱动：高波动 = 信息流快 = 冷却窗口短
function getBaseCooldown(rangePct: number, is0DTE: boolean): number {
  let base: number;
  if (rangePct >= 0.65) base = 5;        // 高波动：5 分钟
  else if (rangePct >= 0.45) base = 8;   // 中波动：8 分钟
  else base = 12;                         // 低波动：12 分钟

  // 非 0DTE 节奏更慢，冷却放大
  if (!is0DTE) base = Math.round(base * 1.5);

  return base;
}
```

如果 rangePct 不可用（非 0DTE 场景），使用 VIX 作为代理：

```typescript
// VIX 驱动（备选）
if (vix >= 25) base = 5;         // 高恐慌 = 高波动
else if (vix >= 18) base = 8;   // 正常
else base = 12;                   // 低波动
```

### 6.4 scoreChangeThreshold 动态化

```typescript
// 用组内最近 N 笔交易的 finalScore 标准差来归一化
// 如果分数总是在 -10 到 -12 之间，Δ=2 就很显著
// 如果分数在 -5 到 -20 之间剧烈波动，Δ=5 才算显著
const recentScores = getRecentGroupScores(group, 10); // 最近 10 笔
const scoreStd = standardDeviation(recentScores);
const scoreChangeThreshold = Math.max(3.0, scoreStd * 1.5);
// 下限 3.0 防止在极低波动时过于敏感
```

如果历史数据不足，退化到固定值 5.0。

## 七、与现有机制的交互

### 7.1 与 0DTE 冷却规则的关系

0DTE 已有的冷却（规则 #14）：首笔 0 冷却 → 2-4笔 ≥1min → 5笔+ ≥3min

这是**频率控制**——控制单位时间内的总交易笔数。
方向感知冷却是**质量控制**——控制是否重复相同的市场判断。

两者应该**并行生效**，取更严格者：

```typescript
// 频率冷却（0DTE 规则 #14）
const frequencyCooldownPassed = (now >= cooldownUntil);

// 方向感知冷却（本方案）
const readiness = calculateReentryReadiness(...);
const directionCooldownPassed = (readiness >= reentryThreshold);

// 两者都通过才允许
const canEnter = frequencyCooldownPassed && directionCooldownPassed;
```

### 7.2 与 smartReverse 的关系

smartReverse 在 Phase B 决定方向（call/put）。方向感知冷却在 Phase C 基于这个方向做竞争过滤。

时序：
```
Phase B: evaluateIdleSymbol()
  → 计算 finalScore
  → detectRegime() 决定是否反向
  → 生成 intent（含最终方向 CALL/PUT）
  → 返回候选人

Phase C: scoringAuction()
  → 组内排名（|finalScore| 降序）
  → 对每个组的 top 候选人，计算 reentryReadiness
    → 比较 intent.metadata.optionDirection 与 recentExits[group].exitDirection
  → 通过/跳过
```

这意味着 smartReverse 的翻转会自然地被方向感知冷却捕获。如果 smartReverse 让方向从 PUT 翻为 CALL，方向感知冷却会识别为"方向变化"，给予高准备度。

### 7.3 与组内互斥的关系

当前：同组有 HOLDING 状态的标的时，新候选人被阻止。
方向感知冷却是在**退出后**的保护，不影响持仓期间的互斥。

两者互补：
- 持仓期间：互斥保护（不变）
- 退出之后：方向感知冷却（新机制）

### 7.4 与跨组并发入场保护的关系

跨组 3 分钟保护仍然生效。方向感知冷却只作用于**组内**（同 correlation group），不影响跨组逻辑。

## 八、预期效果量化

### 8.1 基于 3/19-3/20 数据的模拟

**不使用方向感知冷却（实际结果）**：
- 15 笔 REV_BEAR，PnL = -$394
- 其中 3 笔方向错误贡献 -$767
- 7 笔锁利不足贡献 +$182

**使用方向感知冷却的预估**：
- 同方向持续日（3/20 全天 CALL），每组仅首笔通过 → 约 4-5 笔交易（而非 11 笔）
- 避免了后期低质量重复交易：减亏约 $340-$500
- 但也损失了方向正确的重复交易：损失盈利约 $150-$200
- **净效果预估：+$150 ~ +$300**

### 8.2 在不同市场状态下的预期行为

| 市场状态 | 方向变化频率 | 冷却效果 | 交易频率变化 |
|---------|------------|---------|------------|
| 单向极端日（如 3/20） | 极低 | 强冷却，大幅减少交易 | -60% ~ -70% |
| 震荡日（方向频繁切换） | 高 | 弱冷却，允许跟随切换 | -10% ~ -20% |
| 正常趋势日 | 中等 | 中等冷却 | -30% ~ -40% |

这个特性很好——**在最危险的单向极端日自动减少交易频率，在方向变化的日子保持灵活**。

---

## 九、风险与缓解

### 9.1 方向抖动风险

**风险**：评分在阈值边缘抖动，导致方向来回切换（call→put→call→put），每次切换都通过冷却检查，频繁交易。

**缓解**：
1. timeFactor 提供自然保护（至少需要几分钟时间流逝才能贡献足够分数）
2. 即使方向翻转，reentryReadiness 仍需 ≥ 0.50，纯翻转只贡献 0.35
3. 可以增加一个**最短间隔硬限制**（如 2 分钟），作为所有因子之外的底线保护

```typescript
const MIN_REENTRY_INTERVAL_MS = 2 * 60 * 1000; // 2 分钟硬底线
if (now - lastExitTime < MIN_REENTRY_INTERVAL_MS) {
  return; // 无论准备度多高都不允许
}
```

### 9.2 过滤掉好交易的风险

**风险**：在同方向持续日中，过滤掉了方向正确的重复交易（如 3/20 的 #97 +$76, #98 +$74）。

**缓解**：
1. 从数据看，同方向持续日中，后续交易的期望值为负（3/20 后 10 笔 PnL = -$365）
2. 损失少数好交易是可接受的代价
3. 如果市场确实在同方向持续且有利可图，timeFactor 会在足够时间后自然放行

### 9.3 参数过拟合风险

**风险**：5 因子 + 动态参数的组合可能在训练数据上表现好但泛化差。

**缓解**：
1. 因子权重不动态化（保持稳定）
2. 只有 3 个参数动态化（threshold, baseCooldown, scoreChangeThreshold），且每个的动态范围有上下限
3. 初始权重选择基于第一性原理（方向最重要 → 最高权重），而非数据拟合

## 十一、总结

### 核心创新

从**"固定时间冷却"**进化为**"状态感知动态冷却"**，用五因子评分模型替代硬编码时间：

```
冷却不是"等 10 分钟"，而是"等市场状态变化足够"。
方向翻转是最强的状态变化信号，但不是唯一的。
```

### 第一性原理总结

1. **冷却 = 信息等待窗口**，不是惩罚
2. **信息量 = 市场状态变化量**，可以多维度度量
3. **方向变化是最大的单一信息增量**，但分数变化、时间流逝、盈亏反馈也贡献信息
4. **动态参数应由可观测的市场特征驱动**（波动率、当日 PnL），而非人为设定
5. **在最危险的时刻（同向持续日）自动收紧，在机会丰富时（方向切换）自动放松**
