## 二、与 v2.1 分析的交叉验证

| 市场环境 | mktScore < -39（极端熊） | mktScore = -1.3（近中性） |

| 灾难来源 | 入场太晚 + 止损宽 | 动量减速后重入 + 止损宽 |

### 2.2 但存在关键区别

| 区别 | 熊市 (3/19-3/20) | 牛市 (3/23) |
|------|-----------------|-------------|
| 灾难时段 | 16:00+ EST（临近收盘） | ~11:12 EST（盘中） |
| 灾难原因 | Theta 加速衰减 + 无容错时间 | **动量反转** + 追高入场 |
| intraScore 信号 | intra 从负转正 = 窗口关闭 | intra 仍高但 slope 已转负 |
| 简单修复 | 16:00 后禁止 0DTE ✓ | 时间限制不够——需要 slope 检查 |

**核心洞察**: "16:00 后禁止 0DTE"只能修复熊市场景的尾盘灾难，无法修复牛市场景的盘中动量反转亏损。需要更通用的"动量减速检测"机制。

## 三、intraScore 的结构性缺陷：水位 vs 流速

### 3.1 问题诊断

intraScore 衡量的是"今天涨/跌了多少"（水位），而不是"现在还在涨/跌吗"（流速）。

以 3/23 #5 NVDA 为例：
- 15:12 入场时 intraScore = +17.2（"今天涨了很多"）
- 但此时 NVDA 价格已从日内高点回落
- fastMo slope = -0.0025（"现在正在跌"）
- intraScore 依然高 ≠ 当前还在涨

类似 3/19 #95 NVDA：
- 16:31 入场时 intraScore = +0.36（"日内已从 -2.84 反弹转正"）
- 但此时均值回归动能已耗尽
- 表面看 intra > 0 = "有多头力量"，实际 = "反弹已结束"

### 3.2 第一性原理：水位 vs 流速

| 概念 | 类比 | 交易含义 |
|------|------|---------|
| **水位高** | 水库水位高 | 今天到目前为止涨了很多 |
| **流速正** | 水正在流入 | 当前价格正在上涨 |
| **水位高 + 流速正** | 水库在涨且还在进水 | 动量持续 → 入场有效 |
| **水位高 + 流速负** | 水库虽高但正在放水 | 动量减速 → 入场危险 |
| **水位低 + 流速正** | 水库低位但在补水 | 均值回归进行中 → 反向入场有效 |
| **水位低 + 流速负** | 水库还在泄水 | 趋势继续 → 反向入场极危险 |

当前系统只看水位（intraScore），不看流速（slope）。这就是为什么**在水位高但流速已转负时，还会入场做多**。

### 3.3 对两种市场环境的统一解释

**熊市反向入场**:
- 正确入场：mktScore 极端负 + intraScore 负（正在回归） → "水位低 + 流速正在转正"
- 错误入场：mktScore 极端负 + intraScore 已转正 → "水位已回升 + 流速趋缓"
- v2.1 建议的 "intra > 0 禁止反向入场" 本质上是在检查"流速是否还有"

**牛市顺势入场**:
- 正确入场：mktScore 中性 + intraScore 正且在增长 → "水位涨 + 流速正"
- 错误入场：mktScore 中性 + intraScore 正但 slope 转负 → "水位虽高但流速已转负"
- 需要的检查：slope > 0（"还在涨"而不仅仅是"今天涨了"）

**统一原则**: 无论牛市还是熊市，入场条件除了方向判断外，还需要**确认当前仍有对应方向的边际动量**。

| intraScore | 不限 | **< 0**（均值回归进行中） | v2.1 #93(intra=-2.84,+$235) vs #95(intra=+0.36,-$280) |
| 时间窗口 | 不限 | **16:00 EST 前** | v2.1 3/19-3/20 数据：16:00后亏损集中 |
| 分歧度 | > 0.3 | **0.5 ~ 1.0** | v2.1 分歧度分层分析 |
| 标的限制 | 无 | **SPY/QQQ 需分歧度 > 0.7** | v2.1 SPY 反向 5 笔全亏 |

### 4.2 牛市顺势入场（BULL，mktScore > -10）

| 条件 | 当前 | 建议 | 依据 |
|------|------|------|------|
| intraScore | > 0 | > 0 **且 slope > 0** | 3/23 #5#6：intra高但slope负→亏损 |
| 重入限制 | 方向感知冷却 | 方向感知冷却 **+ slope 检查** | fastMo 已有 slope，需提前到首次入场 |
| 同标的重入 | 无特别限制 | **同标的同方向重入需更高 finalScore** | 3/23 #4(NVDA +$210) → #5(NVDA -$252) |

### 4.3 通用入场增强

| 增强 | 说明 | 预期效果 |
|------|------|---------|
| slope 检查前置 | 将 fastMo 的 slope 检查从 re-entry only 扩展到所有入场 | 过滤动量减速后的首次入场 |
| 方向感知冷却 | 已实施，五因子评分模型 | 过滤同方向低质量重复入场 |
| 锁利强化 | trail 触发 10%→8%，回撤 12%→6% | 方向对时多保留利润 |
