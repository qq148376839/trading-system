# 组内竞价轮换与冷却升级分析

## 问题

2026-03-12 交易日出现同标的快速重入：
- TSLA #1 平仓 → 12秒后 TSLA #2 入场 → 当日最大亏损 -$109
- QQQ 599P 平仓 → 13秒后 QQQ 597P 入场 → 继续亏损 -$36

**根因**：非0DTE 冷却期默认仅 3 分钟，且基于标的的 `lastExitTime`，冷却期太短导致同标的快速重入追高。

## 当前竞价保护机制

| 屏障 | 时长 | 作用域 |
|------|------|--------|
| 冷却期 | 非0DTE 默认 3 分钟 | 单标的 |
| 同组互斥 | 持仓期间 | 组内不同标的 |
| Floor 连锁 | 30 分钟 | 跨组 |
| 跨组并发入场 | 3 分钟 | 跨组 |

**缺口**：冷却期过短 + 不区分盈亏退出 + 无连亏升级机制。

## 方案对比

### 方案 A：排除上次胜出者（简单但粗糙）
- 退出 QQQ → 下次竞价排除 QQQ → 只看 IWM/SPY/TQQQ
- 优点：简单
- 缺点：只排除 1 轮太短，排除 N 轮可能错过机会

### 方案 B：组内轮换冷却
- 退出 QQQ → QQQ 组内冷却 N 分钟
- 已有基础设施（cooldownMinutes），但粒度不够

### 方案 C：组内冷却 + 亏损升级（采用）

| 退出类型 | 冷却时间 | 说明 |
|----------|----------|------|
| 盈利退出 | 10 分钟 | 防止追高重入 |
| 亏损退出 | 20 分钟 | 给市场充分反转时间 |
| 连亏 ≥2 次 | 当日禁止 | 该标的当日不再交易 |

## 实现范围

### 修改文件
- `api/src/services/strategy-scheduler.service.ts`：`evaluateIdleSymbol` 方法中非0DTE冷却逻辑

### 不需要修改
- 0DTE 冷却逻辑（已有完善的升级机制）
- Phase C/D 竞价流程（冷却在 Phase B 生效，自然排除标的）
- 前端（冷却参数硬编码在策略逻辑中，与现有 0DTE 冷却一致）

## 逻辑对照

### 修改前（非0DTE）
```typescript
cooldownMinutes = strategyConfig?.latePeriod?.cooldownMinutes ?? 3;
```

### 修改后（非0DTE）
```typescript
if (consecLosses >= 2) {
  // 连亏2次：当日禁止该标的
  return null; // 直接排除，不进入候选
} else {
  const lastPnL = Number(cancelCtx?.lastTradePnL ?? 0);
  if (!isNaN(lastPnL) && lastPnL > 0) {
    cooldownMinutes = 10; // 盈利退出冷却
  } else {
    cooldownMinutes = 20; // 亏损退出冷却
  }
}
```
