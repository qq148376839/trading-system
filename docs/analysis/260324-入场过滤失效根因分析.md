## 三、根因分析

### 根因 1: decelRatio 只有下界没有上界

**设计意图**: decelRatio = |后半段slope| / |前半段slope|
- decel < 0.5 → 动量减速过半 → 拒绝
- decel ≥ 0.5 → 动量还在 → 放行

**实际问题**: decel=47 意味着后半段斜率是前半段的 **47倍**。

这发生在以下场景：
```
60秒窗口的前30秒: 价格几乎不动 (slope ≈ 0.0025)
60秒窗口的后30秒: 价格突然暴涨 (slope ≈ 0.12)
→ decel = 0.12 / 0.0025 = 48
```

**第一性原理**: 极端加速（decel >> 5）在开盘初期代表的是 **冲量爆发的尾端信号**，而不是"动量健康"。一个 47 倍加速是物理上不可持续的 — 如同火箭的最后一级推进，加速度最大时恰恰是燃料即将耗尽的时刻。

**修复方向**: decelRatio 应有上界。decel ∈ [0.5, 5.0] 为健康区间；decel > 5.0 应触发警告或拒绝。

### 根因 2: IntraScore 极值过滤的检查指标错位

**设计意图**: 当 |intraScore| > 15 时，要求 slope 强度够大 + 动量未减速。

**实际问题**: 在开盘冲量场景中，slope 和 decel 都是 **结构性虚高** 的：
- 60s slope=0.12 → 轻松通过 > 0.001 阈值（差 120 倍）
- decel=47 → 轻松通过 > 0.7 阈值（差 67 倍）

**第一性原理**: 极值过滤的本意是在 intraScore 极高时检测"动量是否还在"，但它检查的两个指标（slope 和 decel）在开盘冲量场景中恰恰都是最高的。这等于在火箭发射时问"引擎还在推吗" — 答案永远是 yes，但你真正需要问的是"燃料还剩多少"。

**真正需要检查的**: 不是"当前还在涨吗"（slope > 0），而是：
1. **价格距离近期高点多远？** — 如果当前价 = 5分钟最高价，说明你在追顶
2. **涨幅相对于开盘移动的占比** — 如果 60s 内的涨幅占开盘总涨幅的 >50%，说明是冲量末段
3. **成交量是否在放大但价格上涨减缓** — 经典的顶部信号（但需要额外数据）

### 根因 3: 冲量守卫不考虑时间维度

**设计意图**: 如果开盘至今移动超过 1.5 ATR → 拦截。

**实际问题**: TSLA 17分钟移动 0.72 ATR。如果维持这个速率，一天将移动 **0.72 × (390/17) = 16.5 ATR** — 这是极端异常的速率，但因为绝对移动量只有 0.72 ATR，守卫未触发。

**第一性原理**: 移动的"危险性"不仅取决于幅度，还取决于速率。0.72 ATR 在一整天确实正常，但在 17 分钟内完成则是极端速率。

**修复方向**: 增加速率维度：
```
moveATR_per_hour = moveATR / (minutesSinceOpen / 60)
= 0.72 / (17/60) = 2.54 ATR/hour
```
如果 moveATR_per_hour > 阈值（例如 2.0）→ 额外提高入场门槛。

### 根因 4 (结构性): 禁入窗口的批次释放效应

**设计意图**: 开盘15分钟禁止入场，等市场稳定后再交易。

**实际问题**: 禁入窗口期间，所有标的持续累积分数。窗口结束的瞬间：
1. 累积了15分钟涨幅的标的得分极高
2. 竞价系统选出得分最高的标的
3. 得分最高的 = 涨得最多的 = 最可能在顶部附近

这是一个 **系统性逆向选择** — 禁入窗口越长，释放后第一笔交易越容易追高。

**第一性原理**: 信息在禁入窗口期间是在积累的（价格在变动），但决策被延迟了。当决策被释放时，它使用的是"过去15分钟的累积信息"，但市场已经部分 price-in 了这些信息。

**修复方向**: 窗口结束后不应立即以正常阈值入场，而应有**渐进式阈值放松**：
```
禁入结束后 0-2 分钟: finalScore 需 ≥ 15 (1.5× 正常阈值)
禁入结束后 2-5 分钟: finalScore 需 ≥ 12 (1.2× 正常阈值)
禁入结束后 5+ 分钟: 恢复正常阈值 10
```

## 五、对 260324-入场优化分析 的反馈

**P0-A (intraScore 极值过滤)**: 已部署但失效。原因是检查指标（slope 强度 + decelRatio 持续性）在开盘冲量场景中结构性虚高，无法区分"健康加速"和"冲量尾端"。

**P1-A (5分钟 longSlope)**: 已部署但在此场景无用。17分钟连涨导致 longSlope 也是强正，方向检查通过。

**开盘冲量守卫**: 已部署但阈值过松。0.72 ATR < 1.5 ATR 未触发，但 0.72 ATR/17min 的速率实际上是极端的。

**总结**: 三道新防线全部被开盘冲量的结构性特征绕过。修复方向是增加对 "加速度过高" 和 "禁入释放后的渐进阈值" 的检查。
