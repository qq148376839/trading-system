# 测试环境搭建指南

## 架构

同一份代码，通过独立 Docker Compose 文件实现环境隔离：

```
生产环境                          测试环境
├── trading-postgres    (内部)    ├── trading-postgres-test (内部)
├── trading-app         :3001    ├── trading-app-test      :3003
└── futu-bridge         :8765    └── (通过宿主机 IP 访问生产 futu-bridge)
```

- 独立 DB volume（`postgres_data_test`）
- 独立 Docker network（`trading-network-test`）
- 前端顶部橙色横幅标识 "TEST ENVIRONMENT"

## 首次搭建

### 1. 创建环境配置文件

```bash
# 根目录
cp .env.test.example .env.test
# 编辑 .env.test，填入测试 DB 密码

# API 目录
cp api/.env.test.example api/.env.test
# 编辑 api/.env.test，填入模拟盘 LongPort key（作为 fallback）
```

### 2. 启动测试环境

```bash
docker compose -p trading-test \
  -f docker-compose.test.yml \
  --env-file .env.test \
  up -d --build
```

### 3. 同步配置数据（需要生产环境运行中）

```bash
bash scripts/sync-test-db.sh
```

同步 7 张配置表：`system_config`, `admin_users`, `capital_allocations`, `strategies`, `trading_rules`, `watchlist`, `stock_blacklist`

### 4. 更新模拟盘 API Key

访问 `http://<NAS_IP>:3003/config`，手动更新：
- `longport_app_key`
- `longport_app_secret`
- `longport_access_token`

## 日常操作

### 启动/停止

```bash
# 启动
docker compose -p trading-test \
  -f docker-compose.test.yml \
  --env-file .env.test \
  up -d

# 停止
docker compose -p trading-test \
  -f docker-compose.test.yml \
  --env-file .env.test \
  down

# 重建（代码更新后）
docker compose -p trading-test \
  -f docker-compose.test.yml \
  --env-file .env.test \
  up -d --build
```

### 重新同步配置

策略配置变更后，重新运行同步脚本：

```bash
bash scripts/sync-test-db.sh
```

脚本幂等（先 TRUNCATE 再导入），同步后需重新设置模拟盘 key。

### 查看日志

```bash
# 测试环境日志
docker logs -f trading-app-test

# 测试 DB
docker exec -it trading-postgres-test psql -U trading_user -d trading_db
```

## 前端环境标识

- 环境变量 `NEXT_PUBLIC_ENV_MODE=test` 在 Docker 构建时注入
- `AppLayout.tsx` 检测到 test 模式后显示固定橙色横幅
- 横幅不可关闭（安全功能，防止误操作）
- 生产环境不设置此变量，横幅不显示
