## 一、执行摘要

4 月 1 日完成的方向系统 + 退出机制重构，在 4 月 2 日实盘中产生了**灾难性结果**：

| 指标 | 数值 | 评级 |
|------|------|------|
| 日净盈亏 | **-$826.27** | CRITICAL |
| 胜率 | 2/6 = 33.3% | POOR |
| 盈亏比 | **0.03:1** | CATASTROPHIC |
| 平均盈利 | +$12.36/笔 | 几乎无意义 |
| 平均亏损 | -$212.75/笔 | 每笔亏 17 倍于盈利 |
| 熔断触发 | 是（$936 >= $862 阈值） | — |
| 方向偏置 | 5/6 笔 BEAR | SEVERE BIAS |

**核心结论**: 系统存在结构性缺陷，不是参数微调能解决的问题。需要从根本机制上大修。

## 二、根因分析（Root Cause Analysis）

### RCA-1: 方向系统信噪比崩塌（CRITICAL）

**现象**: 5/6 笔交易做空，4 笔在 1-3 分钟内触发止损，方向完全错误。

**根因链**:
```
timeWindowAdjustment 被移除 (权重 20% → 0%)
        ↓
intradayScore 权重提升到 80%（原 60%）
        ↓
intradayScore 内部: 1m动量 45% + VWAP 20% + SPX日内 35%
        ↓
BTC/USD 去噪（权重归零），多元化信号源减少
        ↓
方向判断几乎完全依赖短周期噪声信号
        ↓
高 VIX 环境下 1 分钟动量信号频繁翻转
        ↓
系统持续发出看跌信号 → 市场反向 → 连续止损
```

**定量分析**:
- 重构前: 方向得分 = 大盘 20% + 分时 60% + 时间窗 20%（三足鼎立）
- 重构后: 方向得分 = 大盘 20% + 分时 **80%**（分时一家独大）
- 分时内部: 去掉 BTC 15% + USD 15%，将权重全部集中到底层标的和 SPX
- **结果**: 信号源从 5 个有效维度缩减为 3 个，且高度相关（个股动量 + VWAP + SPX 都反映同一个短期方向）

**证据**: 4 月 2 日入场得分 -17.0, -11.4, -22.3, -12.5, -11.0 — 全部指向看跌，但标的在入场后立即反弹。这不是随机噪声，是方向系统被短期反转信号系统性误导。

---

### RCA-2: 盈亏结构性错配（CRITICAL）

**现象**: 盈亏比 0.03:1 — 赢了赚零钱，输了亏大钱。

**根因链**:
```
Trailing Stop 利润锁定台阶设计:
  第一档: 峰值 ≥ 8% → 保底 2%
  第二档: 峰值 ≥ 12% → 保底 6%
        ↓
0DTE gamma 极高 → 价格瞬间冲到 8-12% → 触发第一/二档
        ↓
随后价格快速回落 → 利润跌到保底线 → 立即平仓
        ↓
实际止盈: +2.2%, +3.2%（保底线附近出场）
        ↓
而止损: -20% ~ -24%（FINAL 阶段标准止损 20%）
        ↓
盈亏比 = 2.7% / 23.0% ≈ 0.12:1（即使按百分比也极度失衡）
```

**定量对比**:

| 场景 | 触发条件 | 实际执行 | 问题 |
|------|---------|---------|------|
| 盈利退出 | 峰值 8% → 保底 2% | +2.2%, +3.2% | 保底线太低，利润大量回吐 |
| 亏损退出 | FINAL 止损 -20% | -22.8%, -23.7%, -24.2%, -22.4% | 0DTE 滑点导致实际亏损超过阈值 |
| **不对称度** | | **盈利/亏损 ≈ 1:8** | 需 8 倍胜率才能盈亏平衡 |

**关键洞察**: 当前利润锁定台阶是为多日期权设计的（缓慢移动），不适配 0DTE 高 gamma 场景。0DTE 需要更激进的保底线——第一档的 2% 保底在 0DTE 环境下基本等于 "把利润全部吐回去"。

---

### RCA-3: 同标的连续亏损无有效制动（HIGH）

**现象**: TSLA 在 50 分钟内交易 3 次，2 次止损亏 $440+。

**根因**:
- 策略 10 冷却期规则（规则 #14）: 首笔 0 冷却 / 2-4 笔 1min / 5+ 笔 3min
- 1 分钟冷却对 0DTE 完全不够 — 方向信号在 1 分钟内不会实质性改变
- `maxDailyTradesPerUnderlying` 后端已实现但**当天未配置启用**
- 方向系统持续给出相同看跌信号 → 冷却期过后再次入场 → 再次止损

---

### RCA-4: 高 VIX 环境无防护（HIGH）

**现象**: VIX 因子 1.23-1.35，期权 premium 贵但方向更不确定。

**根因**:
- VIX 因子仅调整入场阈值（`directionalScoreMin × vixFactor`）
- 基础阈值 5（aggressive 模式）× 1.3 = 仅 6.5 分
- 实际入场得分 -11 ~ -22，远超阈值 — VIX 调整完全不够
- **缺少**: 高 VIX 时的仓位缩减、0DTE 暂停、或 premium 上限检查

---

### RCA-5: Theta Bleed 检测存在管辖盲区（MEDIUM）

**审计发现 E3 的实盘印证**:
- Theta bleed 仅在 PnL ∈ [-10%, 4%) 时生效
- Stop loss 在 PnL ≤ -20% 时触发
- **[-10%, -20%] 区间无任何退出机制**
- 当天交易虽然跌穿太快没暴露此问题，但在慢跌场景下将形成 "温水煮青蛙"

## 五、策略大修建议方案

### Phase 1: 紧急止血（立即执行）

> 目标: 防止下一个交易日继续大亏

1. **配置 `maxDailyTradesPerUnderlying`** — 设为 2，防止同标的连续亏损
2. **修复 C1 不可达代码** — 删除 `strategy-scheduler.service.ts:4756`
3. **考虑暂停策略 10** — 直到 Phase 2 完成。当前配置在实盘中已证明负期望值

### Phase 2: 利润锁定重设计（盈亏结构修复）

> 目标: 盈亏比从 0.03:1 提升到 ≥ 0.5:1

**方案 A: 0DTE 专用利润锁定台阶**
```
当前台阶（多日期权思维）:          建议 0DTE 台阶（高 gamma 适配）:
  8% → 保底 2%                      6% → 保底 3.5%
  12% → 保底 6%                     10% → 保底 6.5%
  18% → 保底 12%                    15% → 保底 10%
  30% → 保底 22%                    25% → 保底 18%
  50% → 保底 38%                    40% → 保底 30%
```

核心改变:
- 更低触发阈值（0DTE 涨到 8% 已经很好）
- 更高保底比例（保底 / 峰值 ≥ 55%，而非当前的 25%）
- 减少利润回吐空间

**方案 B: 0DTE 使用时间衰减止盈替代台阶**
- 入场后持仓每过 N 分钟，止盈线自动收紧
- 适配 theta 衰减的物理特性

### Phase 3: 方向系统重构（信号质量修复）

> 目标: 胜率从 33% 提升到 ≥ 50%

核心问题: 当前 80% intradayScore 权重使方向判断过度依赖短期噪声信号。

**建议改动**:

| 维度 | 当前 | 建议 | 理由 |
|------|------|------|------|
| 大盘:分时 权重 | 20:80 | **30:70** 或 **35:65** | 增加宏观信号权重，减少噪声敏感度 |
| BTC/USD 权重 | 0%/0%（去噪） | **评估恢复 5-10%** | 完全去除减少了信号多元化 |
| Gap 信号 | 加法(±15) | **改为乘法权重** | 与其他信号保持一致性（R7） |
| 多窗口趋势 | sign=0 无显式处理 | **添加盘整状态处理** | 避免震荡市默认行为不可预测（R5） |
| 入场阈值 | 5（aggressive） | **提高到 8-10** | 过滤低质量信号 |
| 首小时折扣 | 0.85（降 15%） | **移除或缩小到 0.93** | 开盘首小时波动大但方向不明，不应降低门槛 |

**信号多元化检查清单**:
- [ ] 恢复 BTC/USD 低权重（5%）作为非相关信号源
- [ ] 或替换为其他非相关指标（如行业轮动、期权 put/call ratio）
- [ ] 清理废弃的 `calculateTimeWindowAdjustment()` 代码
- [ ] 删除 BTC/USD 无用计算（当前权重 0 仍在执行）

### Phase 4: 高 VIX 防护体系

> 目标: 高波动环境自动降低风险暴露

| 防护层 | 触发条件 | 动作 |
|--------|---------|------|
| VIX 告警 | VIX 因子 > 1.2 | 日志警告，缩减张数 50% |
| VIX 限制 | VIX 因子 > 1.4 | 0DTE 策略暂停入场 |
| Premium 上限 | 单张 > $3.00（0DTE） | 拒绝入场 |
| 冷却期升级 | VIX 因子 > 1.2 | 同标的冷却 ×2 |

### Phase 5: 退出盲区修复

> 目标: 消除 [-10%, -20%] 管辖真空

**方案**: 扩大 theta bleed 检测范围
```
当前: BLEED_PNL_LOWER = -10%  →  建议: -18%
```
与 STOP_LOSS (-20%) 重叠 2%，确保无死角。

或: 添加**时间止损兜底** — 持仓超过 N 分钟且 PnL < -8%，强制退出。

## 八、结论

当前策略存在两个**独立的结构性缺陷**，任何一个都足以导致负期望值：

1. **利润锁定台阶不适配 0DTE** — 即使方向全对，2% 保底 vs 20% 止损 = 需 91% 胜率才能盈亏平衡
2. **方向系统信噪比崩塌** — 80% 权重压在短期噪声信号上，高 VIX 环境下连续误判

两个缺陷叠加产生了 0.03:1 的灾难性盈亏比。**建议立即暂停策略 10，按 Phase 1-5 顺序修复后，以最小仓位重新验证。**
