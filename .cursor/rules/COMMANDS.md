# 斜杠命令速查表（Slash Commands Reference）

## 🎯 快速激活角色

使用斜杠命令可以快速激活对应的角色，提高对话效率。

## 📋 命令列表

### 👨‍💻 开发角色

**命令**: `/dev` 或 `/developer`

**用途**: 代码编写、功能实现、问题修复

**示例**:
```
/dev 帮我实现一个订单查询功能
/dev 修复这个Bug
/developer 优化这段代码的性能
```

**激活的规则**:
- `developer.md` - 开发角色定义
- `coding-standards.md` - 编码规范
- `api-design.md` - API设计规范
- `architecture.md` - 架构规范

---

### 🔍 审核角色

**命令**: `/review` 或 `/reviewer`

**用途**: 代码审查、质量检查、规范验证

**示例**:
```
/review 审查这段代码
/reviewer 检查这个函数的代码质量
/review 帮我看看这个实现是否符合规范
```

**激活的规则**:
- `reviewer.md` - 审核角色定义
- `coding-standards.md` - 编码规范
- `architecture.md` - 架构规范
- `common.md` - 安全规范

---

### 🧪 测试角色

**用途**: 测试用例编写、质量保证、Bug验证

**示例**:
```

/test 帮我验证这个Bug是否已修复
```

**激活的规则**:
- `tester.md` - 测试角色定义
- `testing.md` - 测试规范
- `coding-standards.md` - 编码规范

---

### 📋 产品经理角色

**示例**:

**激活的规则**:
- `product-manager.md` - 产品经理模式

---

### 📝 项目总结角色 ⭐ 新增

**命令**: `/summarize` 或 `/summarizer`

**用途**: 对话总结、文档整理、导航文件更新

**示例**:
```
/summarize 总结这次对话的内容，整理到文档中
/summarizer 更新项目进度，整理最新文档
/summarize 将这次修复整理到文档并更新导航文件
```

**激活的规则**:
- `project-summarizer.md` - 项目总结角色定义

**执行的操作**:
1. 总结对话内容和成果
2. 整理内容到 `docs/` 目录的适当位置
3. 同步更新 `CHANGELOG.md`、`PROJECT_STATUS.md`、`README.md`、`CODE_MAP.md`

## 💡 使用技巧

### 1. 命令可以单独使用
```
/dev
```
单独使用命令会激活角色，等待后续指令。

### 2. 命令后直接跟需求
```
/dev 帮我实现一个订单查询功能
```
命令和需求可以写在同一行，更高效。

### 3. 命令不区分大小写
```
/DEV 帮我实现功能
/dev 帮我实现功能
/Developer 帮我实现功能
```
以上三种写法都可以。

### 4. 传统方式仍支持
即使不使用斜杠命令，提及关键词也会自动激活：
- "开发"、"编写代码" → 激活开发角色
- "审查"、"检查" → 激活审核角色
- "测试"、"测试用例" → 激活测试角色
- "需求"、"功能" → 激活产品经理角色
- "总结"、"整理文档" → 激活项目总结角色

## 🔄 命令对比

| 激活方式 | 优点 | 缺点 |
|---------|------|------|
| **斜杠命令** | ✅ 明确、快速、一致 | 需要记忆命令 |
| **关键词** | ✅ 自然、灵活 | 可能误激活 |

**推荐**: 使用斜杠命令，更明确、更快速。
